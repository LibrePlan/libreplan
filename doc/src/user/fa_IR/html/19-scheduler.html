زمان‌بند برای زمان‌بندی پویای مشاغل طراحی شده است. با استفاده از Spring Framework Quartz scheduler توسعه یافته است.

برای استفاده موثر از این زمان‌بند، ابتدا باید مشاغل (مشاغل Quartz) که باید زمان‌بندی شوند ایجاد شوند. سپس، این مشاغل می‌توانند به پایگاه داده اضافه شوند، زیرا تمام مشاغل برای زمان‌بندی در پایگاه داده ذخیره می‌شوند.

هنگامی که زمان‌بند شروع می‌کند، مشاغل برای زمان‌بندی یا لغو زمان‌بندی را از پایگاه داده می‌خواند و آن‌ها را به‌طور مناسب زمان‌بندی یا حذف می‌کند. پس از آن، مشاغل می‌توانند به‌صورت پویا با استفاده از رابط کاربری زمان‌بندی شغل اضافه، به‌روزرسانی یا حذف شوند.

Note

زمان‌بند هنگامی که برنامه وب LibrePlan شروع می‌شود آغاز به کار می‌کند و هنگامی که برنامه متوقف می‌شود توقف می‌کند.

Note

این زمان‌بند فقط از عبارات cron برای زمان‌بندی مشاغل پشتیبانی می‌کند.

معیارهایی که زمان‌بند برای زمان‌بندی یا حذف مشاغل هنگام شروع استفاده می‌کند به شرح زیر است:

برای تمام مشاغل:

  • زمان‌بندی
    • شغل دارای اتصال‌دهنده است و اتصال‌دهنده فعال است و شغل مجاز به زمان‌بندی است.
    • شغل دارای اتصال‌دهنده نیست و مجاز به زمان‌بندی است.
  • حذف
    • شغل دارای اتصال‌دهنده است و اتصال‌دهنده فعال نیست.
    • شغل دارای اتصال‌دهنده است و اتصال‌دهنده فعال است، اما شغل مجاز به زمان‌بندی نیست.
    • شغل دارای اتصال‌دهنده نیست و مجاز به زمان‌بندی نیست.

Note

مشاغل در صورتی که در حال اجرا باشند نمی‌توانند دوباره زمان‌بندی یا لغو زمان‌بندی شوند.

نمای فهرست زمان‌بندی شغل

نمای فهرست زمان‌بندی شغل به کاربران امکان می‌دهد:

  • یک شغل جدید اضافه کنند.
  • یک شغل موجود را ویرایش کنند.
  • یک شغل را حذف کنند.
  • یک فرآیند را به‌صورت دستی شروع کنند.

افزودن یا ویرایش شغل

از نمای فهرست زمان‌بندی شغل کلیک کنید:

  • ایجاد برای افزودن یک شغل جدید، یا
  • ویرایش برای تغییر شغل انتخاب‌شده.

هر دو عمل یک فرم شغل ایجاد/ویرایش را باز می‌کنند. فرم ویژگی‌های زیر را نمایش می‌دهد:

  • فیلدها:
    • گروه شغل: نام گروه شغل.
    • نام شغل: نام شغل.
    • عبارت cron: یک فیلد فقط‌خواندنی با دکمه ویرایش برای باز کردن پنجره ورودی عبارت cron.
    • نام کلاس شغل: یک فهرست کشویی برای انتخاب شغل (یک شغل موجود).
    • اتصال‌دهنده: یک فهرست کشویی برای انتخاب اتصال‌دهنده. این اجباری نیست.
    • زمان‌بندی: یک چک‌باکس برای نشان دادن اینکه آیا این شغل زمان‌بندی شود.
  • دکمه‌ها:
    • ذخیره: برای ذخیره یا به‌روزرسانی یک شغل هم در پایگاه داده و هم در زمان‌بند. سپس کاربر به نمای فهرست زمان‌بندی شغل بازگردانده می‌شود.
    • ذخیره و ادامه: مانند "ذخیره" است، اما کاربر به نمای فهرست زمان‌بندی شغل بازگردانده نمی‌شود.
    • لغو: هیچ چیز ذخیره نمی‌شود و کاربر به نمای فهرست زمان‌بندی شغل بازگردانده می‌شود.
  • و یک بخش راهنما درباره نحو عبارت cron.

پاپ‌آپ عبارت cron

برای وارد کردن صحیح عبارت cron، از یک فرم پاپ‌آپ عبارت cron استفاده می‌شود. در این فرم، می‌توانید عبارت cron دلخواه را وارد کنید. همچنین راهنما درباره عبارت cron را ببینید. اگر یک عبارت cron نامعتبر وارد کنید، بلافاصله اطلاع‌رسانی می‌شوید.

حذف شغل

روی دکمه حذف کلیک کنید تا شغل هم از پایگاه داده و هم از زمان‌بند حذف شود. موفقیت یا شکست این عمل نمایش داده می‌شود.

شروع دستی شغل

به عنوان جایگزینی برای انتظار برای اجرای شغل طبق برنامه، می‌توانید روی این دکمه کلیک کنید تا فرآیند را مستقیماً شروع کنید. پس از آن، اطلاعات موفقیت یا شکست در یک پنجره پاپ‌آپ نمایش داده می‌شود.
