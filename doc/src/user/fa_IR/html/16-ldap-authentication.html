این صفحه به شما اجازه می‌دهد تا یک اتصال با LDAP برای واگذاری احراز هویت و/یا مجوزدهی برقرار کنید.

این صفحه به چهار ناحیه مختلف تقسیم شده است که در ادامه توضیح داده می‌شوند:

فعال‌سازی

این ناحیه برای تنظیم ویژگی‌هایی استفاده می‌شود که تعیین می‌کنند چگونه LibrePlan از LDAP استفاده می‌کند.

اگر فیلد فعال‌سازی احراز هویت LDAP علامت‌زده شود، LibrePlan هر بار که کاربری سعی می‌کند وارد برنامه شود LDAP را جستجو می‌کند.

علامت‌زدن فیلد استفاده از نقش‌های LDAP به این معنی است که یک نگاشت بین نقش‌های LDAP و نقش‌های LibrePlan برقرار می‌شود. در نتیجه، مجوزهای یک کاربر در LibrePlan به نقش‌هایی که کاربر در LDAP دارد بستگی خواهد داشت.

پیکربندی

این بخش حاوی مقادیر پارامتر برای دسترسی به LDAP است. Base، UserDN و Password پارامترهایی هستند که برای اتصال به LDAP و جستجوی کاربران استفاده می‌شوند. بنابراین، کاربر مشخص‌شده باید مجوز انجام این عملیات را در LDAP داشته باشد. در پایین این بخش، یک دکمه برای بررسی امکان اتصال LDAP با پارامترهای داده‌شده وجود دارد. توصیه می‌شود قبل از ادامه پیکربندی، اتصال را آزمایش کنید.

Note

اگر LDAP شما برای کار با احراز هویت ناشناس پیکربندی شده باشد، می‌توانید ویژگی‌های UserDN و Password را خالی بگذارید.

Tip

در مورد پیکربندی Active Directory (AD)، فیلد Base باید مکان دقیقی باشد که کاربر متصل‌شده در AD قرار دارد.

مثال: ou=organizational_unit,dc=example,dc=org

احراز هویت

اینجا می‌توانید ویژگی موجود در گره‌های LDAP را که در آن نام کاربری داده‌شده باید یافت شود پیکربندی کنید. ویژگی UserId باید با نام ویژگی‌ای که نام کاربری در آن در LDAP ذخیره شده است پر شود.

کادر تیک ذخیره رمزهای عبور در پایگاه داده، هنگامی که علامت‌زده می‌شود، به این معنی است که رمز عبور نیز در پایگاه داده LibrePlan ذخیره می‌شود. به این ترتیب، اگر LDAP آفلاین یا غیرقابل دسترس باشد، کاربران LDAP می‌توانند در برابر پایگاه داده LibrePlan احراز هویت کنند. اگر علامت‌زده نشود، کاربران LDAP فقط می‌توانند در برابر LDAP احراز هویت شوند.

مجوزدهی

این بخش به شما اجازه می‌دهد تا یک استراتژی برای تطبیق نقش‌های LDAP با نقش‌های LibrePlan تعریف کنید. اولین انتخاب استراتژی مورد استفاده است که بسته به پیاده‌سازی LDAP متفاوت است.

استراتژی گروه

هنگامی که از این استراتژی استفاده می‌شود، نشان می‌دهد که LDAP دارای یک استراتژی نقش-گروه است. این به این معنی است که کاربران در LDAP گره‌هایی هستند که مستقیماً زیر یک شاخه‌ای قرار دارند که گروه را نمایندگی می‌کند.

مثال بعدی یک ساختار LDAP معتبر برای استفاده از استراتژی گروه را نشان می‌دهد.

  • ساختار LDAP:

    dc=example,dc=org
|- ou=groups
   |- cn=admins
   |- cn=itpeople
   |- cn=workers
|- ou=people
   |- uid=admin1
   |- uid=it1
   |- uid=it2
   |- uid=worker1
   |- uid=worker2
   |- uid=worker3

در این حالت، هر گروه یک ویژگی خواهد داشت، به عنوان مثال member نام دارد، با فهرست کاربران متعلق به گروه:

  • cn=admins:
    • member: uid=admin1,ou=people,dc=example,dc=org
    • member: uid=it1,ou=people,dc=example,dc=org
  • cn=itpeople:
    • member: uid=it1,ou=people,dc=example,dc=org
    • member: uid=it2,ou=people,dc=example,dc=org
  • cn=workers:
    • member: uid=worker1,ou=people,dc=example,dc=org
    • member: uid=worker2,ou=people,dc=example,dc=org
    • member: uid=worker3,ou=people,dc=example,dc=org

پیکربندی برای این حالت به شرح زیر است:

  • استراتژی جستجوی نقش: Group strategy
  • مسیر گروه: ou=groups
  • ویژگی نقش: member
  • پرسش جستجوی نقش: uid=[USER_ID],ou=people,dc=example,dc=org

و به عنوان مثال، اگر بخواهید برخی نقش‌ها را تطبیق دهید:

  • مدیریت: cn=admins;cn=itpeople
  • خواننده سرویس وب: cn=itpeople
  • نویسنده سرویس وب: cn=itpeople
  • مجاز به خواندن همه پروژه‌ها: cn=admins
  • مجاز به ویرایش همه پروژه‌ها: cn=admins
  • مجاز به ایجاد پروژه: cn=workers

استراتژی ویژگی

هنگامی که یک مدیر تصمیم می‌گیرد از این استراتژی استفاده کند، نشان می‌دهد که هر کاربر یک گره LDAP است و در داخل گره، یک ویژگی وجود دارد که گروه(های) مربوط به کاربر را نمایندگی می‌کند. در این حالت، پیکربندی به پارامتر مسیر گروه نیازی ندارد.

مثال بعدی یک ساختار LDAP معتبر برای استفاده از استراتژی ویژگی را نشان می‌دهد.

  • ساختار LDAP:

    dc=example,dc=org
|- ou=people
   |- uid=admin1
   |- uid=it1
   |- uid=it2
   |- uid=worker1
   |- uid=worker2
   |- uid=worker3

با ویژگی

در این حالت، هر کاربر یک ویژگی خواهد داشت، به عنوان مثال group نام دارد، با نام گروهی که به آن تعلق دارد:

  • uid=admin1:
    • group: admins
  • uid=it1:
    • group: itpeople
  • uid=it2:
    • group: itpeople
  • uid=worker1:
    • group: workers
  • uid=worker2:
    • group: workers
  • uid=worker3:
    • group: workers

Warning

این استراتژی یک محدودیت دارد: هر کاربر می‌تواند تنها به یک گروه تعلق داشته باشد.

پیکربندی برای این حالت به شرح زیر است:

  • استراتژی جستجوی نقش: Property strategy
  • مسیر گروه:
  • ویژگی نقش: group
  • پرسش جستجوی نقش: [USER_ID]

و به عنوان مثال، اگر بخواهید برخی نقش‌ها را تطبیق دهید:

  • مدیریت: admins;itpeople
  • خواننده سرویس وب: itpeople
  • نویسنده سرویس وب: itpeople
  • مجاز به خواندن همه پروژه‌ها: admins
  • مجاز به ویرایش همه پروژه‌ها: admins
  • مجاز به ایجاد پروژه: workers

با شناسه کاربری

می‌توانید حتی از یک راه‌حل جایگزین برای تعیین نقش‌های LibrePlan مستقیماً به کاربران بدون داشتن ویژگی در هر کاربر LDAP استفاده کنید.

در این حالت، مشخص می‌کنید کدام کاربران نقش‌های مختلف LibrePlan را بر اساس uid دارند.

پیکربندی برای این حالت به شرح زیر است:

  • استراتژی جستجوی نقش: Property strategy
  • مسیر گروه:
  • ویژگی نقش: uid
  • پرسش جستجوی نقش: [USER_ID]

و به عنوان مثال، اگر بخواهید برخی نقش‌ها را تطبیق دهید:

  • مدیریت: admin1;it1
  • خواننده سرویس وب: it1;it2
  • نویسنده سرویس وب: it1;it2
  • مجاز به خواندن همه پروژه‌ها: admin1
  • مجاز به ویرایش همه پروژه‌ها: admin1
  • مجاز به ایجاد پروژه: worker1;worker2;worker3

تطبیق نقش

در پایین این بخش، یک جدول با تمام نقش‌های LibrePlan و یک فیلد متنی در کنار هر کدام وجود دارد. این برای تطبیق نقش‌ها است. به عنوان مثال، اگر یک مدیر تصمیم بگیرد که نقش مدیریت LibrePlan با نقش‌های admin و administrators در LDAP تطابق دارد، فیلد متنی باید حاوی: "admin;administrators" باشد. کاراکتر جداکننده نقش‌ها ";" است.

Note

اگر می‌خواهید مشخص کنید که همه کاربران یا همه گروه‌ها یک مجوز دارند، می‌توانید از ستاره (*) به عنوان کاراکتر عام برای ارجاع به آن‌ها استفاده کنید. به عنوان مثال، اگر می‌خواهید همه نقش مجاز به ایجاد پروژه را داشته باشند، تطبیق نقش را به شرح زیر پیکربندی می‌کنید:

  • مجاز به ایجاد پروژه: *
