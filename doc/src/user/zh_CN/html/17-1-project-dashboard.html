项目仪表板是 LibrePlan 的一个视角，包含一组**关键绩效指标（KPI）**，可协助评估项目在以下方面的表现：

  • 工作进度
  • 成本
  • 已分配资源的状态
  • 时间限制

进度绩效指标

计算两个指标：项目进度百分比和任务状态。

项目进度百分比

此图表显示项目的整体进度，并与基于 Gantt 图的预期进度进行比较。

进度由两个条形表示：

  • 当前进度： 根据已采取的测量值得出的当前进度。
  • 预期进度： 根据项目计划，项目此时应达到的进度。

若要查看每个条形的实际测量值，请将鼠标光标悬停在条形上。

整体项目进度使用几种不同的方法估算，因为没有单一、普遍正确的方法：

  • 扩散进度： 这是在项目层级设置为扩散进度的进度类型。在此情况下，无法计算预期值，仅显示当前条形。
  • 依所有任务工时： 所有项目任务的进度取平均值以计算整体值。这是考量分配给每个任务工时数的加权平均值。
  • 依关键路径工时： 属于任何项目关键路径的任务进度取平均值以获得整体值。这是考量每个相关任务总分配工时的加权平均值。
  • 依关键路径持续时间： 属于任何关键路径的任务进度使用加权平均值取平均，但这次考量的是每个相关任务的持续时间而非分配的工时。

任务状态

饼图显示不同状态的项目任务百分比。定义的状态为：

  • 已完成： 以100%进度值识别的已完成任务。
  • 进行中： 目前正在进行的任务。这些任务的进度值不为0%或100%，或已追踪了一些工作时间。
  • 准备开始： 进度为0%、无追踪工时、所有*结束到开始*依赖任务均已*完成*，且所有*开始到开始*依赖任务均已*完成*或*进行中*的任务。
  • 已阻塞： 进度为0%、无追踪工时，且前置依赖任务既不*进行中*也不在*准备开始*状态的任务。

成本指标

计算几个*实获值管理*成本指标：

  • CV（成本差异）： 当前时刻*实获值曲线*与*实际成本曲线*之间的差异。正值表示收益，负值表示损失。

  • ACWP（已执行工作的实际成本）： 当前时刻项目中已追踪的总工时数。

  • CPI（成本绩效指数）： *实获值/实际成本*比率。

    • > 0 表示项目低于预算。

资源

若要从资源角度分析项目，提供两个比率和一个直方图。

已完成任务估算偏差直方图

此直方图计算分配给项目任务的工时数与实际投入工时数之间的偏差。

偏差以百分比计算所有已完成任务，并在直方图中表示计算出的偏差。纵轴显示每个偏差区间内的任务数量。动态计算六个偏差区间。

加班比率

此比率汇总分配给项目任务的资源超载情况。使用公式计算：加班比率 = 超载 / (负载 + 超载)。

  • = 0 为有利，表示资源未超载。
  • > 0 为不利，表示资源超载。

可用性比率

此比率汇总当前分配给项目的资源的空闲容量。因此，它衡量资源在不超载的情况下接受更多分配的可用性。计算公式为：可用性比率 = (1 - 负载/容量) * 100

  • 可能的值介于0%（完全分配）到100%（未分配）之间。

时间

包含两个图表：项目任务完成时间偏差的直方图和截止日期违反的饼图。

任务完成提前或延后

此计算确定项目任务的计划结束时间与实际结束时间之间的天数差异。计划完成日期取自 Gantt 图，实际完成日期取自任务的最后一次追踪工时。

任务完成的延后或提前以直方图表示。纵轴显示提前/延后天数差值对应横坐标天数区间的任务数量。计算六个动态任务完成偏差区间。

  • 负值表示提前完成。
  • 正值表示延后完成。

截止日期违反

本节计算与项目截止日期（若已设置）的差距。此外，饼图显示符合截止日期的任务百分比。图表中包含三种类型的值：

  • 未设置截止日期的任务百分比。
  • 实际结束日期晚于其截止日期的已结束任务百分比。实际结束日期取自任务的最后一次追踪工时。
  • 实际结束日期早于其截止日期的已结束任务百分比。
