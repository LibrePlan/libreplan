工作报告使人们能够监控资源投入到其被分配的任务的工时。

程序允许用户配置用于输入投入工时的新表单，指定他们希望在这些表单中出现的字段。这允许纳入员工执行任务的报告，以及监控员工活动。

在用户可以为资源添加条目之前，他们必须定义至少一种工作报告类型。此类型定义报告的结构，包括添加到其中的所有行。用户可以在系统中根据需要创建任意数量的工作报告类型。

工作报告类型

工作报告由一系列对整个报告通用的字段和一组具有每行定义的字段的特定值的工作报告行组成。例如，资源和任务对所有报告通用。但是，可能有其他新字段，例如「事件」，这在所有报告类型中都不是必需的。

工作报告类型

工作报告类型的管理允许用户配置这些类型并添加新的文本字段或可选标签。在编辑工作报告类型的第一个选项卡中，可以为必填属性（无论它们适用于整个报告还是在行级别指定）配置类型，并添加新的可选字段。

所有工作报告中必须出现的必填字段如下：

  • 名称和代码： 工作报告类型名称及其代码的识别字段。
  • 日期： 报告日期的字段。
  • 资源： 出现在报告或工作报告行上的员工或机器。
  • 项目元素： 所执行工作被归因到的项目元素的代码。
  • 工时管理： 决定要使用的工时归因政策，可以是：
    • 根据分配工时： 工时根据分配的工时进行归因。
    • 根据开始和完成时间： 工时根据开始和完成时间计算。
    • 根据工时数和开始/完成范围： 允许差异，工时数优先。

用户可以向报告添加新字段：

  • 标签类型： 用户可以请求系统在完成工作报告时显示标签。例如，客户标签类型，如果用户希望在每份报告中输入执行工作的客户。
  • 自由字段： 可以在工作报告中自由输入文本的字段。

创建具有个性化字段的工作报告类型

用户可以配置日期、资源和项目元素字段出现在报告标题中，这意味着它们适用于整个报告，或者它们可以添加到每一行。

最后，可以使用「附加文本」和「标签类型」字段，分别在工作报告标题或每行中，向现有字段添加新的附加文本字段或标签。用户可以在「管理附加字段和标签」选项卡中配置这些元素的输入顺序。

工作报告列表

一旦要纳入系统的报告格式配置完成，用户可以根据对应工作报告类型中定义的结构，在创建的表单中输入详细信息。为此，用户需要遵循以下步骤：

  • 从工作报告类型列表中，点击与所需报告相关联的「新工作报告」按钮。
  • 程序然后根据为类型提供的配置显示报告。见下图。

基于类型的工作报告结构

  • 选择报告所示的所有字段：
    • 资源： 如果选择了标题，资源只显示一次。或者，对于报告的每一行，都需要选择一个资源。
    • 任务代码： 正在分配工作报告的任务的代码。与其他字段类似，如果字段在标题中，则在报告行中输入一次或根据需要输入多次。
    • 日期： 报告或每行的日期，取决于标题或行是否已配置。
    • 工时数： 项目中的工作工时数。
    • 开始和完成时间： 工作的开始和完成时间，以计算确定的工作时数。此字段仅在工时分配政策为「根据开始和完成时间」和「根据工时数和开始/完成范围」的情况下出现。
    • 工时类型： 使用户能够选择工时类型，例如「正常」、「加班」等。
  • 点击「保存」或「保存并继续」。
