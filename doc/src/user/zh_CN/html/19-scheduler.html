调度器设计用于动态调度作业，使用 Spring Framework Quartz scheduler 开发。

若要有效使用此调度器，必须先创建需要调度的作业（Quartz 作业）。然后，这些作业可以添加至数据库，因为所有待调度的作业均存储在数据库中。

调度器启动时，会从数据库读取待调度或待取消调度的作业，并据此调度或移除它们。之后，可以使用 作业调度 用户界面动态添加、更新或移除作业。

Note

调度器在 LibrePlan 网络应用程序启动时开始运行，在应用程序停止时停止。

Note

此调度器仅支持使用 cron 表达式 来调度作业。

调度器启动时，用于调度或移除作业的判断标准如下：

对于所有作业：

  • 调度
    • 作业具有 连接器，且 连接器 已激活，且作业允许被调度。
    • 作业没有 连接器，且允许被调度。
  • 移除
    • 作业具有 连接器，且 连接器 未激活。
    • 作业具有 连接器，且 连接器 已激活，但作业不允许被调度。
    • 作业没有 连接器，且不允许被调度。

Note

若作业正在执行中，则无法重新调度或取消调度。

作业调度列表视图

作业调度列表 视图允许用户：

  • 添加新作业。
  • 编辑现有作业。
  • 移除作业。
  • 手动启动过程。

添加或编辑作业

从 作业调度列表 视图，点击：

  • 创建 以添加新作业，或
  • 编辑 以修改所选的作业。

两个操作都将打开创建/编辑 作业表单。表单 显示以下属性：

  • 字段：
    • 作业组： 作业组的名称。
    • 作业名称： 作业的名称。
    • Cron 表达式： 只读字段，含 编辑 按钮，可打开 cron 表达式 输入窗口。
    • 作业类名： 用于选择作业（现有作业）的 下拉列表。
    • 连接器： 用于选择连接器的 下拉列表。此项非必填。
    • 调度： 复选框，用于指示是否调度此作业。
  • 按钮：
    • 保存： 在数据库和调度器中保存或更新作业，并返回 作业调度列表视图。
    • 保存并继续： 与「保存」相同，但不返回 作业调度列表视图。
    • 取消： 不保存任何内容，并返回 作业调度列表视图。
  • 以及关于 cron 表达式语法的提示部分。

Cron 表达式弹出窗口

若要正确输入 cron 表达式，请使用 cron 表达式 弹出表单。在此表单中，您可以输入所需的 cron 表达式。另请参阅关于 cron 表达式 的提示。若输入无效的 cron 表达式，将立即收到通知。

移除作业

点击 移除 按钮，从数据库和调度器中删除作业。此操作的成功或失败结果将予以显示。

手动启动作业

除了等待作业按调度执行外，您也可以点击此按钮直接启动过程。之后，成功或失败的信息将显示在 弹出窗口 中。
