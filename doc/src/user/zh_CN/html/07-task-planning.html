任务规划

LibrePlan 中的规划是贯穿整个用户指南描述的过程，其中关于项目和资源分配的章节尤为重要。本章描述在项目和甘特图正确配置后的基本规划程序。

工作规划视图

与公司概览一样，项目规划视图根据正在分析的信息分为几个视图。特定项目可用的视图包括：

  • 规划视图
  • 资源负载视图
  • 项目列表视图
  • 高级分配视图

规划视图

规划视图结合了三种不同的角度：

  • 项目规划： 项目规划以甘特图形式显示在程序的右上方。此视图允许用户临时移动任务、在任务之间指定依赖关系、定义里程碑以及建立限制。
  • 资源负载： 资源负载视图位于界面右下方，根据分配显示资源可用性，而非任务的分配情况。此视图中显示的信息如下：
    • 紫色区域： 表示资源负载低于其容量的 100%。
    • 绿色区域： 表示由于资源被规划用于另一个项目，资源负载低于 100%。
    • 橙色区域： 表示由于当前项目，资源负载超过 100%。
    • 黄色区域： 表示由于其他项目，资源负载超过 100%。
  • 图表视图和挣值指标： 可以从「挣值」选项卡查看。生成的图表基于挣值技术，指标针对项目的每个工作日计算。计算的指标包括：
    • BCWS（计划工作的预算成本）： 截至某日期的计划工时数的累积时间函数。在任务的计划开始时为 0，在结束时等于计划工时总数。与所有累积图表一样，它将始终增加。任务的函数是截至计算日期每日分配的总和。此函数在所有时间都有值，前提是已分配资源。
    • ACWP（已执行工作的实际成本）： 截至某日期工作报告中报告工时的累积时间函数。此函数在任务的第一份工作报告日期之前只有 0 的值，随着时间流逝和工作报告工时的增加，其值将继续增加。在最后一份工作报告日期之后没有值。
    • BCWP（已执行工作的预算成本）： 包含将任务进度乘以估计完成任务所需工作量的结果值的累积时间函数。随着时间流逝，此函数的值增加，进度值也增加。进度乘以所有任务的估计工时总数。BCWP 值是正在计算的任务的值的总和。进度在配置时被汇总。
    • CV（成本差异）： CV = BCWP - ACWP
    • SV（排程差异）： SV = BCWP - BCWS
    • BAC（完工预算）： BAC = max (BCWS)
    • EAC（完工估算）： EAC = (ACWP / BCWP) * BAC
    • VAC（完工差异）： VAC = BAC - EAC
    • ETC（完工尚需估算）： ETC = EAC - ACWP
    • CPI（成本绩效指数）： CPI = BCWP / ACWP
    • SPI（排程绩效指数）： SPI = BCWP / BCWS

在项目规划视图中，用户可以执行以下操作：

  • 分配依赖关系： 在任务上右键点击，选择「添加依赖关系」，然后将鼠标指针拖曳到应分配依赖关系的任务。
    • 要更改依赖关系类型，在依赖关系上右键点击并选择所需的类型。
  • 创建新里程碑： 点击要在其之前添加里程碑的任务，然后选择「添加里程碑」选项。里程碑可以通过用鼠标指针选择里程碑并将其拖曳到所需位置来移动。
  • 在不干扰依赖关系的情况下移动任务： 在任务主体上右键点击并将其拖曳到所需位置。如果没有限制或依赖关系被违反，系统将更新资源对任务的每日分配，并将任务放置在选定的日期。
  • 分配限制： 点击相关任务并选择「任务属性」选项。将出现一个带有可修改的「限制」字段的弹出窗口。限制可能与依赖关系冲突，这就是为什么每个项目都指定依赖关系是否优先于限制。可以建立的限制包括：
    • 尽早开始： 表示任务必须尽快开始。
    • 不早于： 表示任务不得在某个日期之前开始。
    • 在特定日期开始： 表示任务必须在特定日期开始。

规划视图还提供几个作为查看选项的程序：

  • 缩放级别： 用户可以选择所需的缩放级别。有几个缩放级别：年度、四个月、月度、每周和每日。
  • 搜索筛选器： 用户可以根据标签或标准筛选任务。
  • 关键路径： 由于使用 Dijkstra 算法计算图上的路径，关键路径被实现。可以通过点击查看选项中的「关键路径」按钮来查看。
  • 显示标签： 使用户能够查看分配给项目中任务的标签，可以在屏幕上查看或打印。
  • 显示资源： 使用户能够查看分配给项目中任务的资源，可以在屏幕上查看或打印。
  • 打印： 使用户能够打印正在查看的甘特图。

资源负载视图

资源负载视图提供资源列表，其中包含产生工作量的任务或标准的列表。每个任务或标准都以甘特图形式显示，以便可以看到负载的开始和结束日期。根据资源的负载是高于还是低于 100% 显示不同的颜色：

  • 绿色： 负载低于 100%
  • 橙色： 100% 负载
  • 红色： 负载超过 100%

特定项目的资源负载视图

如果将鼠标指针放在资源的甘特图上，将显示员工的负载百分比。
