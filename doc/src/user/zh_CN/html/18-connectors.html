连接器是 LibrePlan 的客户端应用程序，可用于与（网络）服务器通信以获取数据、处理数据并加以存储。目前有三种连接器：JIRA 连接器、Tim Enterprise 连接器和电子邮件连接器。

配置

连接器在使用前必须正确配置。可从「主要设置」屏幕的「连接器」选项卡进行配置。

连接器屏幕包含：

  • 下拉列表： 可用连接器的列表。
  • 属性编辑屏幕： 所选连接器的属性编辑表单。
  • 测试连接按钮： 用于测试与连接器连接的按钮。

从连接器的下拉列表中选择要配置的连接器，屏幕将显示所选连接器的属性编辑器表单。在属性编辑器表单中，您可以根据需要更改属性值，并使用「测试连接」按钮测试您的配置。

Note

属性已设有默认值。最重要的属性是「已激活」。默认值为「N」，表示除非您将值更改为「Y」并保存更改，否则不会使用该连接器。

JIRA 连接器

JIRA 是一套问题与项目跟踪系统。

JIRA 连接器是一个应用程序，可用于向 JIRA 网络服务器请求 JIRA 问题数据并处理响应。请求以 JIRA 标签为基础。在 JIRA 中，标签可用于对问题进行分类。请求的结构如下：检索所有以此标签名称分类的问题。

连接器接收响应（即问题），并将其转换为 LibrePlan 的「项目元素」和「工时表」。

*JIRA 连接器*在使用前必须正确配置。

配置

从「主要设置」屏幕选择「连接器」选项卡。在连接器屏幕中，从下拉列表选择 JIRA 连接器，随即显示属性编辑器屏幕。

在此屏幕中，您可以配置以下属性值：

  • 已激活： Y/N，表示是否要使用 JIRA 连接器。默认值为「N」。
  • 服务器 URL： JIRA 网络服务器的绝对路径。
  • 用户名和密码： 授权用的用户凭据。
  • JIRA 标签：逗号分隔的标签列表或 URL： 您可以输入标签 URL 或逗号分隔的标签列表。
  • 工时类型： 工作工时的类型。默认值为「默认」。

Note

JIRA 标签： 目前 JIRA 网络服务器不支持提供所有可用标签的列表。作为变通方案，我们开发了一个简单的 PHP 脚本，在 JIRA 数据库中执行简单的 SQL 查询以获取所有不重复的标签。您可以将此 PHP 脚本作为「JIRA 标签 URL」使用，或在「JIRA 标签」字段中以逗号分隔文本输入所需的标签。

最后，点击「测试连接」按钮，测试是否可以连接至 JIRA 网络服务器，以及您的配置是否正确。

同步

在项目窗口的「一般数据」下，您可以开始将项目元素与 JIRA 问题进行同步。

点击「与 JIRA 同步」按钮以开始同步。

  • 若这是第一次，将显示一个弹出窗口（含自动完成的标签列表）。在此窗口中，您可以选择要同步的标签，并点击「开始同步」按钮开始同步过程，或点击「取消」按钮取消。
  • 若标签已同步，JIRA 屏幕将显示上次同步日期和标签。在此情况下，不会显示选择标签的弹出窗口，而是直接对该已显示（已同步）的标签开始同步过程。

Note

「项目」与「标签」之间的关系是一对一。一个「项目」只能与一个标签同步。

Note

成功（重新）同步后，信息将写入数据库，JIRA 屏幕将更新为最新的同步日期和标签。

（重新）同步分两个阶段执行：

  • 第一阶段： 同步项目元素，包括进度分配和测量值。
  • 第二阶段： 同步工时表。

Note

若第一阶段失败，将不会执行第二阶段，也不会将任何信息写入数据库。

Note

成功或失败的信息将显示在弹出窗口中。

同步成功完成后，结果将显示在「项目详情」屏幕的「工作分解结构（WBS 任务）」选项卡中。此用户界面与标准 WBS 相比有两项变更：

  • 「任务总工时」列不可修改（只读），因为同步是单向的。任务工时只能在 JIRA 网络服务器中更新。
  • 「代码」列显示 JIRA 问题键值，同时也是 JIRA 问题的超链接。若要前往该键值（JIRA 问题）的文档，请点击所需的键值。

调度

JIRA 问题的重新同步也可以通过调度器执行。前往「作业调度」屏幕，在该屏幕中可配置 JIRA 作业以执行同步。作业会搜索数据库中最后同步的标签，并据此重新同步。另请参阅调度器手册。

Tim Enterprise 连接器

Tim Enterprise 是 Aenova 的荷兰产品，是一款基于网络的应用程序，用于管理项目和任务上所花费的时间。

Tim 连接器是一个可与 Tim Enterprise 服务器通信的应用程序，用于：

  • 导出工作人员（用户）在项目中所花费的所有工时，这些工时可以在 Tim Enterprise 中登记。
  • 导入工作人员（用户）的所有班表，以有效规划资源。

*Tim 连接器*在使用前必须正确配置。

配置

从「主要设置」屏幕选择「连接器」选项卡。在连接器屏幕中，从下拉列表选择 Tim 连接器，随即显示属性编辑器屏幕。

在此屏幕中，您可以配置以下属性值：

  • 已激活： Y/N，表示是否要使用 Tim 连接器。默认值为「N」。
  • 服务器 URL： Tim Enterprise 服务器的绝对路径。
  • 用户名和密码： 授权用的用户凭据。
  • 导出至 Tim 的工时表天数： 您要导出工时表的过去天数。
  • 从 Tim 导入班表的天数： 您要导入班表的未来天数。
  • 生产力系数： 有效工作时间的百分比。默认值为「100%」。
  • 要导入班表的部门 ID： 逗号分隔的部门 ID。

最后，点击「测试连接」按钮，测试是否可以连接至 Tim Enterprise 服务器，以及您的配置是否正确。

导出

在项目窗口的「一般数据」下，您可以开始将工时表导出至 Tim Enterprise 服务器。

输入「Tim 产品代码」，然后点击「导出至 Tim」按钮以开始导出。

Tim 连接器会连同产品代码一起添加以下字段：

  • 工作人员/用户的全名。
  • 工作人员执行任务的日期。
  • 投入的工作量或在任务上所花费的工时。
  • 一个选项，指示 Tim Enterprise 是否应更新登记或插入新的登记。

Tim Enterprise 的响应只包含记录 ID（整数）的列表。由于响应列表只包含与请求字段无关的数字，这使得判断发生了什么问题变得困难。若列表中所有条目均不包含「0」值，则视为导出请求（在 Tim 中登记）成功。否则，包含「0」值的条目的导出请求即为失败。因此，您无法看到哪个请求失败，因为列表条目只包含值「0」。判断此问题的唯一方法是检查 Tim Enterprise 服务器上的日志文件。

Note

成功导出后，信息将写入数据库，Tim 屏幕将更新为最新的导出日期和产品代码。

Note

成功或失败的信息将显示在弹出窗口中。

调度导出

导出过程也可以通过调度器执行。前往「作业调度」屏幕，在该屏幕中可配置 Tim 导出作业。作业会搜索数据库中最后导出的工时表，并据此重新导出。另请参阅调度器手册。

导入

导入班表只能通过调度器运作。由于用户无需提供任何输入，因此没有为此设计用户界面。前往「作业调度」屏幕，配置 Tim 导入作业。作业会遍历连接器属性中配置的所有部门，并导入每个部门的所有班表。另请参阅调度器手册。

导入时，Tim 连接器在请求中添加以下字段：

  • 期间： 您要导入班表的期间（起始日期－截止日期）。这可作为筛选条件提供。
  • 部门： 您要导入班表的部门。部门可配置。
  • 您感兴趣的字段（如人员信息、班表类别等），Tim 服务器应在其响应中包含这些字段。

导入响应包含以下字段，足以管理 LibrePlan 中的例外日期：

  • 人员信息： 姓名和网络名称。
  • 部门： 工作人员所在的部门。
  • 班表类别： 工作人员出勤/缺勤（Aanwzig/afwezig）的信息，以及工作人员缺勤时的原因（LibrePlan 例外类型）。
  • 日期： 工作人员出勤/缺勤的日期。
  • 时间： 出勤/缺勤的开始时间，例如 08:00。
  • 时长： 工作人员出勤/缺勤的小时数。

将导入响应转换为 LibrePlan 的「例外日期」时，需考虑以下翻译：

  • 若班表类别包含名称「Vakantie」，将翻译为「RESOURCE HOLIDAY」（资源假日）。
  • 班表类别「Feestdag」将翻译为「BANK HOLIDAY」（法定假日）。
  • 其他所有类别，如「Jus uren」、「PLB uren」等，应手动添加至「日历例外日期」。

此外，在导入响应中，班表每天分为两或三个部分：例如早班、下午班和晚班。然而，LibrePlan 每天只允许一种「例外类型」。Tim 连接器负责将这些部分合并为一种例外类型。即以时长最长的班表类别作为有效的例外类型，但总时长为这些类别部分的所有时长之和。

与 LibrePlan 不同，在 Tim Enterprise 中，工作人员请假时的总时长表示工作人员在该总时长内不可用。然而，在 LibrePlan 中，若工作人员请假，总时长应为零。Tim 连接器也负责处理此翻译。

电子邮件连接器

电子邮件是一种从发件人向一位或多位收件人交换数字消息的方法。

电子邮件连接器可用于设置简单邮件传输协议（SMTP）服务器的连接属性。

*电子邮件连接器*在使用前必须正确配置。

配置

从「主要设置」屏幕选择「连接器」选项卡。在连接器屏幕中，从下拉列表选择电子邮件连接器，随即显示属性编辑器屏幕。

在此屏幕中，您可以配置以下属性值：

  • 已激活： Y/N，表示是否要使用电子邮件连接器。默认值为「N」。
  • 协议： SMTP 协议的类型。
  • 主机： SMTP 服务器的绝对路径。
  • 端口： SMTP 服务器的端口。
  • 发件地址： 消息发件人的电子邮件地址。
  • 用户名： SMTP 服务器的用户名。
  • 密码： SMTP 服务器的密码。

最后，点击「测试连接」按钮，测试是否可以连接至 SMTP 服务器，以及您的配置是否正确。

编辑电子邮件模板

在项目窗口的「配置」下，然后选择「编辑电子邮件模板」，您可以修改消息的电子邮件模板。

您可以选择：

  • 模板语言：
  • 模板类型：
  • 电子邮件主题：
  • 模板内容：

您需要指定语言，因为网络应用程序将以用户在其偏好设置中所选择的语言发送电子邮件。您需要选择模板类型。类型是用户角色，表示此电子邮件只会发送给具有所选角色（类型）的用户。您需要设置电子邮件主题。主题是消息话题的简短摘要。您需要设置电子邮件内容。这是您要发送给用户的任何信息。消息中还有一些可使用的关键字，网络应用程序将解析这些关键字并以新值替换。

调度电子邮件

发送电子邮件只能通过调度器执行。前往「配置」，然后选择「作业调度」屏幕。在该屏幕中，您可以配置电子邮件发送作业。作业会获取电子邮件通知列表，收集数据，并将其发送至用户的电子邮件。另请参阅调度器手册。

Note

成功或失败的信息将显示在弹出窗口中。
