Критерии
########

.. contents::

Критерии — это элементы, используемые в программе для категоризации как ресурсов, так и задач. Задачи требуют определённых критериев, а ресурсы должны соответствовать этим критериям.

Пример использования критериев: ресурсу присваивается критерий «сварщик» (то есть ресурс соответствует категории «сварщик»), а задача требует критерия «сварщик» для выполнения. Следовательно, при распределении ресурсов по задачам с использованием общего распределения (в отличие от конкретного распределения) будут рассматриваться работники с критерием «сварщик». Для получения дополнительной информации о различных типах распределения обратитесь к главе о распределении ресурсов.

Программа позволяет выполнять несколько операций с критериями:

*   Администрирование критериев
*   Назначение критериев ресурсам
*   Назначение критериев задачам
*   Фильтрация сущностей на основе критериев. Задачи и элементы проектов могут быть отфильтрованы по критериям для выполнения различных операций в программе.

В этом разделе будет объяснена только первая функция — администрирование критериев. Два типа распределения будут рассмотрены позднее: распределение ресурсов в главе «Управление ресурсами» и фильтрация в главе «Планирование задач».

Администрирование критериев
============================

Доступ к администрированию критериев осуществляется через меню администрирования:

.. figure:: images/menu.png
   :scale: 50

   Вкладки меню первого уровня

Конкретная операция для управления критериями — *Управление критериями*. Эта операция позволяет просмотреть список критериев, доступных в системе.

.. figure:: images/lista-criterios.png
   :scale: 50

   Список критериев

Доступ к форме создания/редактирования критерия можно получить, нажав кнопку *Создать*. Для редактирования существующего критерия нажмите значок редактирования.

.. figure:: images/edicion-criterio.png
   :scale: 50

   Редактирование критериев

Форма редактирования критериев, как показано на предыдущем изображении, позволяет выполнять следующие операции:

*   **Редактировать название критерия.**
*   **Указать, можно ли одновременно присваивать несколько значений или только одно значение для выбранного типа критерия.** Например, ресурс может соответствовать двум критериям: «сварщик» и «токарь».
*   **Указать тип критерия:**

    *   **Общий:** Критерий, который можно использовать как для машин, так и для сотрудников.
    *   **Сотрудник:** Критерий, который можно использовать только для сотрудников.
    *   **Машина:** Критерий, который можно использовать только для машин.

*   **Указать, является ли критерий иерархическим.** Иногда критерии необходимо рассматривать иерархически. Например, присвоение критерия элементу не приводит автоматически к его присвоению производным от него элементам. Ярким примером иерархического критерия является «местоположение». Например, лицо, обозначенное местоположением «Галисия», также будет принадлежать «Испании».
*   **Указать, авторизован ли критерий.** Это способ деактивации критериев пользователями. После того как критерий был создан и использован в исторических данных, его нельзя изменить. Вместо этого его можно деактивировать, чтобы он не появлялся в списках выбора.
*   **Описать критерий.**
*   **Добавить новые значения.** Поле ввода текста с кнопкой *Новый критерий* расположено во второй части формы.
*   **Редактировать названия существующих значений критериев.**
*   **Перемещать значения критериев вверх или вниз в списке текущих значений критериев.**
*   **Удалять значение критерия из списка.**

Форма администрирования критериев следует поведению форм, описанному во введении, и предлагает три действия: *Сохранить*, *Сохранить и закрыть* и *Закрыть*.
