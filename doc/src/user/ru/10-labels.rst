Метки
#####

.. contents::

Метки — это сущности, используемые в программе для концептуальной организации задач или элементов проекта.

Метки классифицируются в соответствии с типом метки. Метка может принадлежать только одному типу меток; однако пользователи могут создавать множество похожих меток, принадлежащих к разным типам меток.

Типы меток
==========

Типы меток используются для группировки типов меток, которыми пользователи хотят управлять в программе. Вот несколько примеров возможных типов меток:

*   **Клиент:** Пользователи могут быть заинтересованы в маркировке задач, проектов или элементов проекта в связи с клиентом, который их запрашивает.
*   **Область:** Пользователи могут быть заинтересованы в маркировке задач, проектов или элементов проекта в связи с областями, в которых они выполняются.

Администрирование типов меток осуществляется из пункта меню «Администрирование». Здесь пользователи могут редактировать типы меток, создавать новые типы меток и добавлять метки к типам меток. Из этого пункта пользователи могут получить доступ к списку меток.

.. figure:: images/tag-types-list.png
   :scale: 50

   Список типов меток

Из списка типов меток пользователи могут:

*   Создать новый тип меток.
*   Редактировать существующий тип меток.
*   Удалить тип меток вместе со всеми его метками.

Редактирование и создание меток используют одну и ту же форму. Из этой формы пользователь может присвоить имя типу меток, создавать или удалять метки и сохранять изменения. Процедура следующая:

*   Выбрать метку для редактирования или нажать кнопку создания для новой.
*   Система отображает форму с текстовым полем для имени и списком текстовых полей с существующими и назначенными метками.
*   Если пользователи хотят добавить новую метку, они должны нажать кнопку «Новая метка».
*   Система отображает новую строку в списке с пустым текстовым полем, которое пользователи должны отредактировать.
*   Пользователи вводят имя метки.
*   Система добавляет имя в список.
*   Пользователи нажимают «Сохранить» или «Сохранить и продолжить», чтобы продолжить редактирование формы.

Метки
=====

Метки — это сущности, принадлежащие типу меток. Эти сущности можно назначать элементам проекта. Назначение метки элементу проекта означает, что все элементы, дочерние по отношению к этому элементу, унаследуют метку, которой они принадлежат. Наличие назначенной метки означает, что эти сущности можно фильтровать при выполнении поиска:

*   Поиск задач на диаграмме Ганта.
*   Поиск элементов проекта в списке элементов проекта.
*   Фильтры для отчётов.

Назначение меток элементам проекта рассматривается в главе о проектах.
