Панель управления проектом
##########################

.. contents::

Панель управления проектом — это перспектива *LibrePlan*, содержащая набор **KPI (ключевых показателей эффективности)** для оценки производительности проекта по следующим параметрам:

   *   Прогресс работы
   *   Затраты
   *   Статус выделенных ресурсов
   *   Временные ограничения

Показатели прогресса
====================

Рассчитываются два показателя: процент прогресса проекта и статус задач.

Процент прогресса проекта
-------------------------

На этой диаграмме отображается общий прогресс проекта в сравнении с ожидаемым прогрессом согласно диаграмме *Ганта*.

Прогресс представлен двумя столбцами:

   *   *Текущий прогресс:* Текущий прогресс на основе выполненных измерений.
   *   *Ожидаемый прогресс:* Прогресс, которого проект должен был достичь к данному моменту согласно плану проекта.

Для просмотра фактического измеренного значения каждого столбца наведите курсор мыши на столбец.

Общий прогресс проекта оценивается несколькими различными методами, поскольку не существует единого универсально правильного подхода:

   *   **Распределённый прогресс:** Это тип прогресса, установленный в качестве распределённого на уровне проекта. В этом случае нет возможности рассчитать ожидаемое значение, и отображается только текущий столбец.
   *   **По всем часам задач:** Прогресс всех задач проекта усредняется для расчёта общего значения. Это взвешенное среднее, учитывающее количество часов, выделенных для каждой задачи.
   *   **По часам критического пути:** Прогресс задач, принадлежащих любому из критических путей проекта, усредняется для получения общего значения. Это взвешенное среднее, учитывающее общее количество выделенных часов для каждой задействованной задачи.
   *   **По продолжительности критического пути:** Прогресс задач, принадлежащих любому из критических путей, усредняется с использованием взвешенного среднего, но на этот раз с учётом продолжительности каждой задействованной задачи, а не назначенных часов.

Статус задач
------------

На круговой диаграмме показан процент задач проекта в различных состояниях. Определены следующие состояния:

   *   **Завершённые:** Выполненные задачи, определяемые значением прогресса 100%.
   *   **В процессе:** Задачи, которые в настоящее время выполняются. Эти задачи имеют значение прогресса, отличное от 0% или 100%, либо для них зафиксировано рабочее время.
   *   **Готовы к запуску:** Задачи с прогрессом 0%, без зафиксированного времени, все зависимые задачи с типом *FINISH_TO_START* которых *завершены*, а все зависимые задачи с типом *START_TO_START* — *завершены* или *в процессе*.
   *   **Заблокированные:** Задачи с прогрессом 0%, без зафиксированного времени и с предшествующими зависимыми задачами, которые не находятся ни в состоянии *в процессе*, ни в состоянии *готова к запуску*.

Показатели затрат
=================

Рассчитываются несколько показателей управления освоенным объёмом (*Earned Value Management*):

   *   **CV (отклонение по стоимости):** Разница между *кривой освоенного объёма* и *кривой фактических затрат* на текущий момент. Положительные значения указывают на прибыль, отрицательные — на убыток.
   *   **ACWP (фактическая стоимость выполненных работ):** Общее количество часов, зафиксированных в проекте на текущий момент.
   *   **CPI (индекс выполнения стоимости):** Соотношение *Освоенного объёма / Фактических затрат*.

        *   > 100 — благоприятный результат, проект укладывается в бюджет.
        *   = 100 — также благоприятный, затраты соответствуют плану.
        *   < 100 — неблагоприятный, стоимость выполнения работ выше запланированной.
   *   **ETC (оценка до завершения):** Оставшееся время до завершения проекта.
   *   **BAC (бюджет по завершении):** Общий объём работ, выделенный в плане проекта.
   *   **EAC (оценка по завершении):** Прогноз руководителя общей стоимости при завершении проекта на основе *CPI*.
   *   **VAC (отклонение по завершении):** Разница между *BAC* и *EAC*.

        *   < 0 указывает на перерасход бюджета.
        *   > 0 указывает на экономию бюджета.

Ресурсы
=======

Для анализа проекта с точки зрения ресурсов предоставляются два коэффициента и гистограмма.

Гистограмма отклонения оценки для завершённых задач
---------------------------------------------------

Эта гистограмма рассчитывает отклонение между количеством часов, выделенных для задач проекта, и фактическим количеством часов, посвящённых им.

Отклонение рассчитывается в процентах для всех завершённых задач, и рассчитанные отклонения представлены в гистограмме. Вертикальная ось показывает количество задач в каждом интервале отклонения. Шесть интервалов отклонения рассчитываются динамически.

Коэффициент сверхурочной работы
-------------------------------

Этот коэффициент суммирует перегруженность ресурсов, выделенных для задач проекта. Рассчитывается по формуле: **коэффициент сверхурочной работы = перегрузка / (нагрузка + перегрузка)**.

   *   = 0 — благоприятный результат, ресурсы не перегружены.
   *   > 0 — неблагоприятный, ресурсы перегружены.

Коэффициент доступности
-----------------------

Этот коэффициент суммирует свободную мощность ресурсов, в настоящее время выделенных для проекта. Таким образом, он измеряет доступность ресурсов для получения дополнительных назначений без перегрузки. Рассчитывается как: **коэффициент доступности = (1 - нагрузка/мощность) * 100**

   *   Возможные значения — от 0% (полностью назначены) до 100% (не назначены).

Время
=====

Включены две диаграммы: гистограмма для временного отклонения в сроках завершения задач проекта и круговая диаграмма для нарушений дедлайна.

Опережение или задержка завершения задач
-----------------------------------------

Этот расчёт определяет разницу в днях между плановым временем окончания задач проекта и их фактическим временем окончания. Плановая дата завершения берётся из диаграммы *Ганта*, а фактическая дата окончания — из последнего зафиксированного времени для задачи.

Опережение или задержка завершения задач представлена в гистограмме. Вертикальная ось показывает количество задач со значением разницы в днях опережения/задержки, соответствующим интервалу дней на абсциссе. Рассчитываются шесть динамических интервалов отклонения завершения задач.

   *   Отрицательные значения означают завершение раньше срока.
   *   Положительные значения означают завершение позже срока.

Нарушения дедлайна
------------------

В этом разделе рассчитывается запас до дедлайна проекта (при его наличии). Кроме того, на круговой диаграмме показан процент задач, укладывающихся в свои дедлайны. В диаграмму включены три типа значений:

   *   Процент задач без настроенного дедлайна.
   *   Процент завершённых задач с фактической датой окончания позже их дедлайна. Фактическая дата окончания берётся из последнего зафиксированного времени для задачи.
   *   Процент завершённых задач с фактической датой окончания раньше их дедлайна.
