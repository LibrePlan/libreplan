Планирование задач
==================

Планирование в LibrePlan — это процесс, описанный на протяжении всего руководства пользователя, особую важность имеют главы о проектах и назначении ресурсов. В этой главе описываются основные процедуры планирования после того, как проект и диаграммы Ганта были правильно настроены.

.. figure:: images/planning-view.png
   :scale: 35

   Представление планирования работ

Как и в случае обзора компании, представление планирования проекта разделено на несколько представлений в зависимости от анализируемой информации. Представления, доступные для конкретного проекта:

*   Представление планирования
*   Представление загрузки ресурсов
*   Представление списка проектов
*   Представление расширенного назначения

Представление планирования
---------------------------

Представление планирования объединяет три различных перспективы:

*   **Планирование проекта:** Планирование проекта отображается в верхней правой части программы в виде диаграммы Ганта. Это представление позволяет пользователям временно перемещать задачи, устанавливать зависимости между ними, определять вехи и устанавливать ограничения.
*   **Загрузка ресурсов:** Представление загрузки ресурсов, расположенное в нижней правой части экрана, показывает доступность ресурсов на основе назначений, в отличие от назначений задачам. Информация, отображаемая в этом представлении:

    *   **Фиолетовая область:** Указывает на загрузку ресурсов ниже 100% от мощности.
    *   **Зелёная область:** Указывает на загрузку ресурсов ниже 100%, возникающую из-за планирования ресурса на другой проект.
    *   **Оранжевая область:** Указывает на загрузку ресурсов более 100% в результате текущего проекта.
    *   **Жёлтая область:** Указывает на загрузку ресурсов более 100% в результате других проектов.

*   **Представление графика и показатели освоенного объёма:** Их можно просмотреть на вкладке «Освоенный объём». Сгенерированный график основан на методе освоенного объёма, и показатели рассчитываются для каждого рабочего дня проекта. Рассчитываемые показатели:

    *   **BCWS (Budgeted Cost of Work Scheduled — Плановые затраты на запланированные работы):** Кумулятивная временная функция количества часов, запланированных до определённой даты. На плановом начале задачи она будет равна 0 и равна общему количеству запланированных часов в конце. Как и все кумулятивные графики, она всегда возрастает. Функция для задачи — это сумма ежедневных назначений до даты расчёта. Эта функция имеет значения для всех моментов времени при условии назначения ресурсов.
    *   **ACWP (Actual Cost of Work Performed — Фактические затраты на выполненные работы):** Кумулятивная временная функция часов, отражённых в рабочих отчётах до определённой даты. Эта функция будет иметь значение 0 только до даты первого рабочего отчёта по задаче, и её значение будет продолжать увеличиваться по мере течения времени и добавления часов рабочих отчётов. После даты последнего рабочего отчёта значений нет.
    *   **BCWP (Budgeted Cost of Work Performed — Плановые затраты на выполненные работы):** Кумулятивная временная функция, включающая результирующее значение умножения прогресса задачи на объём работ, который, по оценке, потребует задача для завершения. Значения этой функции увеличиваются с течением времени, как и значения прогресса. Прогресс умножается на общее количество расчётных часов для всех задач. Значение BCWP — это сумма значений для рассчитываемых задач. Прогресс суммируется при его настройке.
    *   **CV (Cost Variance — Отклонение по стоимости):** CV = BCWP - ACWP
    *   **SV (Schedule Variance — Отклонение от графика):** SV = BCWP - BCWS
    *   **BAC (Budget at Completion — Бюджет по завершении):** BAC = max (BCWS)
    *   **EAC (Estimate at Completion — Оценка по завершении):** EAC = (ACWP / BCWP) * BAC
    *   **VAC (Variance at Completion — Отклонение по завершении):** VAC = BAC - EAC
    *   **ETC (Estimate to Complete — Оценка до завершения):** ETC = EAC - ACWP
    *   **CPI (Cost Performance Index — Индекс выполнения стоимости):** CPI = BCWP / ACWP
    *   **SPI (Schedule Performance Index — Индекс выполнения графика):** SPI = BCWP / BCWS

В представлении планирования проекта пользователи могут выполнять следующие действия:

*   **Назначение зависимостей:** Щёлкните правой кнопкой мыши задачу, выберите «Добавить зависимость» и перетащите указатель мыши на задачу, которой должна быть назначена зависимость.

    *   Для изменения типа зависимости щёлкните правой кнопкой мыши на зависимость и выберите нужный тип.

*   **Создание новой вехи:** Нажмите задачу, перед которой нужно добавить веху, и выберите опцию «Добавить веху». Вехи можно перемещать, выбирая их указателем мыши и перетаскивая в нужное положение.
*   **Перемещение задач без нарушения зависимостей:** Щёлкните правой кнопкой мыши тело задачи и перетащите его в нужное положение. Если ни ограничения, ни зависимости не нарушаются, система обновит ежедневное назначение ресурсов задаче и разместит задачу на выбранную дату.
*   **Назначение ограничений:** Нажмите соответствующую задачу и выберите опцию «Свойства задачи». Появится всплывающее окно с полем «Ограничения», которое можно изменить. Ограничения могут конфликтовать с зависимостями, поэтому в каждом проекте указывается, имеют ли зависимости приоритет над ограничениями. Устанавливаемые ограничения:

    *   **Как можно раньше:** Указывает, что задача должна начаться как можно скорее.
    *   **Не раньше:** Указывает, что задача не должна начаться раньше определённой даты.
    *   **Начать в определённую дату:** Указывает, что задача должна начаться в определённую дату.

Представление планирования также предлагает несколько процедур, функционирующих как параметры просмотра:

*   **Уровень масштабирования:** Пользователи могут выбрать нужный уровень масштабирования. Есть несколько уровней масштабирования: годовой, четырёхмесячный, месячный, еженедельный и ежедневный.
*   **Фильтры поиска:** Пользователи могут фильтровать задачи на основе меток или критериев.
*   **Критический путь:** В результате использования алгоритма *Дейкстры* для вычисления путей на графах был реализован критический путь. Его можно просмотреть, нажав кнопку «Критический путь» в параметрах просмотра.
*   **Показать метки:** Позволяет пользователям просматривать метки, назначенные задачам в проекте, которые можно просматривать на экране или распечатать.
*   **Показать ресурсы:** Позволяет пользователям просматривать ресурсы, назначенные задачам в проекте, которые можно просматривать на экране или распечатать.
*   **Печать:** Позволяет пользователям распечатать просматриваемую диаграмму Ганта.

Представление загрузки ресурсов
----------------------------------

Представление загрузки ресурсов предоставляет список ресурсов, содержащий список задач или критериев, создающих рабочую нагрузку. Каждая задача или критерий отображаются в виде диаграммы Ганта, чтобы можно было видеть даты начала и окончания нагрузки. Отображается разный цвет в зависимости от того, превышает ли нагрузка ресурса 100% или нет:

*   **Зелёный:** Нагрузка ниже 100%
*   **Оранжевый:** Нагрузка 100%
*   **Красный:** Нагрузка выше 100%

.. figure:: images/resource-load.png
   :scale: 35

   Представление загрузки ресурсов для конкретного проекта

При наведении указателя мыши на диаграмму Ганта ресурса будет показан процент загрузки работника.

------------------------------
