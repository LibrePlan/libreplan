Коннекторы — это клиентские приложения LibrePlan, которые используются для взаимодействия с (веб-)серверами с целью получения данных, их обработки и сохранения. В настоящее время доступны три коннектора: коннектор JIRA, коннектор Tim Enterprise и коннектор электронной почты.

Настройка

Перед использованием коннекторы необходимо правильно настроить. Это можно сделать на экране «Основные настройки» на вкладке «Коннекторы».

Экран коннекторов включает:

  • Раскрывающийся список: Список доступных коннекторов.
  • Экран редактирования свойств: Форма редактирования свойств выбранного коннектора.
  • Кнопка проверки соединения: Кнопка для проверки соединения с коннектором.

Выберите нужный коннектор из раскрывающегося списка. Отобразится форма редактирования свойств выбранного коннектора. В этой форме можно изменить значения свойств по мере необходимости и проверить конфигурацию с помощью кнопки «Проверить соединение».

Note

Свойства настроены со значениями по умолчанию. Наиболее важное свойство — «Активирован». По умолчанию оно установлено в «N». Это означает, что коннектор не будет использоваться, пока вы не измените значение на «Y» и не сохраните изменения.

Коннектор JIRA

JIRA — это система отслеживания задач и управления проектами.

Коннектор JIRA — это приложение, которое позволяет запрашивать данные веб-сервера JIRA по задачам и обрабатывать ответ. Запрос основан на метках JIRA. В JIRA метки используются для категоризации задач. Структура запроса такова: получить все задачи, категоризированные указанной меткой.

Коннектор получает ответ, содержащий задачи, и преобразует их в «Элементы проекта» и «Табели учёта рабочего времени» LibrePlan.

Перед использованием коннектор JIRA необходимо правильно настроить.

Настройка

На экране «Основные настройки» перейдите на вкладку «Коннекторы». На экране коннекторов выберите коннектор JIRA из раскрывающегося списка. После этого откроется экран редактирования свойств.

На этом экране можно настроить следующие значения свойств:

  • Активирован: Y/N — указывает, следует ли использовать коннектор JIRA. Значение по умолчанию — «N».
  • URL сервера: Абсолютный путь к веб-серверу JIRA.
  • Имя пользователя и пароль: Учётные данные пользователя для авторизации.
  • Метки JIRA: список меток через запятую или URL: Можно ввести URL меток или список меток через запятую.
  • Тип часов: Тип рабочих часов. Значение по умолчанию — «Default».

Note

Метки JIRA: В настоящее время веб-сервер JIRA не поддерживает предоставление полного списка доступных меток. В качестве обходного решения был разработан простой PHP-скрипт, выполняющий простой SQL-запрос в базе данных JIRA для получения всех уникальных меток. Можно либо использовать этот PHP-скрипт в качестве «URL меток JIRA», либо ввести нужные метки через запятую в поле «Метки JIRA».

Наконец, нажмите кнопку «Проверить соединение», чтобы убедиться в возможности подключения к веб-серверу JIRA и в корректности настроек.

Синхронизация

В окне проекта, в разделе «Общие данные», можно запустить синхронизацию элементов проекта с задачами JIRA.

Нажмите кнопку «Синхронизировать с JIRA», чтобы начать синхронизацию.

  • Если это первая синхронизация, откроется всплывающее окно с автозаполненным списком меток. В этом окне можно выбрать метку для синхронизации и нажать кнопку «Начать синхронизацию», чтобы запустить процесс, или кнопку «Отмена» для его отмены.
  • Если метка уже синхронизирована, на экране JIRA будут отображены дата последней синхронизации и метка. В этом случае всплывающее окно выбора метки не появится — синхронизация начнётся непосредственно для отображённой (уже синхронизированной) метки.

Note

Связь между «Проектом» и «меткой» является взаимно однозначной. С одним «Проектом» можно синхронизировать только одну метку.

Note

После успешной (пере)синхронизации информация будет записана в базу данных, а экран JIRA обновится с отображением даты последней синхронизации и метки.

(Пере)синхронизация выполняется в два этапа:

  • Этап 1: Синхронизация элементов проекта, включая назначение прогресса и замеры.
  • Этап 2: Синхронизация табелей учёта рабочего времени.

Note

Если Этап 1 завершится неудачей, Этап 2 выполнен не будет, и информация не будет записана в базу данных.

Note

Информация об успехе или неудаче будет отображена во всплывающем окне.

После успешного завершения синхронизации результат отобразится на вкладке «Иерархическая структура работ (задачи WBS)» экрана «Детали проекта». В этом интерфейсе есть два отличия от стандартной WBS:

  • Столбец «Всего часов задачи» недоступен для изменения (только для чтения), поскольку синхронизация является односторонней. Часы задачи можно обновлять только на веб-сервере JIRA.
  • В столбце «Код» отображаются ключи задач JIRA, которые также являются гиперссылками на задачи JIRA. Нажмите нужный ключ, чтобы перейти к документу для этой задачи (задача JIRA).

Планирование

Повторная синхронизация задач JIRA может также выполняться через планировщик. Перейдите на экран «Планирование заданий». На этом экране можно настроить задание JIRA для выполнения синхронизации. Задание выполняет поиск последних синхронизированных меток в базе данных и повторно синхронизирует их. См. также руководство по планировщику.

Коннектор Tim Enterprise

Tim Enterprise — нидерландский продукт компании Aenova. Это веб-приложение для учёта времени, затраченного на проекты и задачи.

Коннектор Tim — это приложение, которое используется для взаимодействия с сервером Tim Enterprise в целях:

  • Экспорта всех часов, отработанных сотрудником (пользователем) над проектом, которые могут быть зарегистрированы в Tim Enterprise.
  • Импорта всех табелей сотрудника (пользователя) для эффективного планирования ресурсов.

Перед использованием коннектор Tim необходимо правильно настроить.

Настройка

На экране «Основные настройки» перейдите на вкладку «Коннекторы». На экране коннекторов выберите коннектор Tim из раскрывающегося списка. После этого откроется экран редактирования свойств.

На этом экране можно настроить следующие значения свойств:

  • Активирован: Y/N — указывает, следует ли использовать коннектор Tim. Значение по умолчанию — «N».
  • URL сервера: Абсолютный путь к серверу Tim Enterprise.
  • Имя пользователя и пароль: Учётные данные пользователя для авторизации.
  • Количество дней табеля для Tim: Количество дней назад, за которые необходимо экспортировать табели.
  • Количество дней табеля из Tim: Количество дней вперёд, за которые необходимо импортировать расписания.
  • Коэффициент производительности: Эффективные рабочие часы в процентах. Значение по умолчанию — «100%».
  • Идентификаторы отделов для импорта расписания: Идентификаторы отделов через запятую.

Наконец, нажмите кнопку «Проверить соединение», чтобы убедиться в возможности подключения к серверу Tim Enterprise и в корректности настроек.

Экспорт

В окне проекта, в разделе «Общие данные», можно запустить экспорт табелей на сервер Tim Enterprise.

Введите «Код продукта Tim» и нажмите кнопку «Экспортировать в Tim», чтобы начать экспорт.

Коннектор Tim добавляет к коду продукта следующие поля:

  • Полное имя сотрудника/пользователя.
  • Дата, когда сотрудник работал над задачей.
  • Трудозатраты или количество часов, отработанных над задачей.
  • Параметр, указывающий, должен ли Tim Enterprise обновить существующую запись или создать новую.

Ответ Tim Enterprise содержит только список идентификаторов записей (целые числа). Это затрудняет определение причины ошибки, поскольку список ответов содержит только числа, не связанные с полями запроса. Считается, что запрос на экспорт (регистрацию в Tim) выполнен успешно, если ни одна из записей списка не содержит значение «0». В противном случае запрос на экспорт завершился неудачей для тех записей, которые содержат значение «0». Определить, какой именно запрос завершился неудачей, невозможно, так как записи списка содержат только значение «0». Единственный способ выяснить это — изучить файл журнала на сервере Tim Enterprise.

Note

После успешного экспорта информация будет записана в базу данных, а экран Tim обновится с отображением даты последнего экспорта и кода продукта.

Note

Информация об успехе или неудаче будет отображена во всплывающем окне.

Планирование экспорта

Процесс экспорта также может выполняться через планировщик. Перейдите на экран «Планирование заданий». На этом экране можно настроить задание экспорта Tim. Задание выполняет поиск последних экспортированных табелей в базе данных и повторно экспортирует их. См. также руководство по планировщику.

Импорт

Импорт расписаний работает только с помощью планировщика. Пользовательский интерфейс для этого не предусмотрен, так как ввод данных от пользователя не требуется. Перейдите на экран «Планирование заданий» и настройте задание импорта Tim. Задание проходит по всем отделам, настроенным в свойствах коннектора, и импортирует все расписания для каждого отдела. См. также руководство по планировщику.

Для импорта коннектор Tim добавляет в запрос следующие поля:

  • Период: Период (дата начала — дата окончания), за который необходимо импортировать расписание. Может использоваться как критерий фильтрации.
  • Отдел: Отдел, для которого необходимо импортировать расписание. Отделы настраиваются.
  • Поля, которые вас интересуют (например, информация о сотруднике, категория расписания и т. д.), которые сервер Tim должен включить в ответ.

Ответ на импорт содержит следующие поля, которых достаточно для управления исключительными днями в LibrePlan:

  • Информация о сотруднике: Имя и сетевое имя.
  • Отдел: Отдел, в котором работает сотрудник.
  • Категория расписания: Информация о присутствии/отсутствии (Aanwzig/afwezig) сотрудника и причина (LibrePlan тип исключения) в случае отсутствия сотрудника.
  • Дата: Дата присутствия/отсутствия сотрудника.
  • Время: Время начала присутствия/отсутствия, например 08:00.
  • Продолжительность: Количество часов присутствия/отсутствия сотрудника.

При преобразовании ответа на импорт в «Исключительный день» LibrePlan учитываются следующие соответствия:

  • Если категория расписания содержит название «Vakantie», она будет переведена как «ОТПУСК РЕСУРСА».
  • Категория расписания «Feestdag» будет переведена как «ПРАЗДНИЧНЫЙ ДЕНЬ».
  • Все остальные категории, такие как «Jus uren», «PLB uren» и т. д., необходимо вручную добавлять в раздел «Исключительные дни календаря».

Кроме того, в ответе на импорт расписание разбито на два или три части за каждый день: например, расписание утром, во второй половине дня и вечером. Однако LibrePlan допускает только один «Тип исключения» в день. Коннектор Tim отвечает за объединение этих частей в один тип исключения. То есть категория расписания с наибольшей продолжительностью считается допустимым типом исключения, но общая продолжительность равна сумме продолжительностей всех частей этой категории.

В отличие от LibrePlan, в Tim Enterprise общая продолжительность в случае отпуска сотрудника означает, что сотрудник недоступен в течение этой продолжительности. Однако в LibrePlan, если сотрудник находится в отпуске, общая продолжительность должна быть равна нулю. Коннектор Tim также обрабатывает это преобразование.

Коннектор электронной почты

Электронная почта — это метод обмена цифровыми сообщениями от отправителя одному или нескольким получателям.

Коннектор электронной почты используется для настройки свойств подключения к серверу Simple Mail Transfer Protocol (SMTP).

Перед использованием коннектор электронной почты необходимо правильно настроить.

Настройка

На экране «Основные настройки» перейдите на вкладку «Коннекторы». На экране коннекторов выберите коннектор электронной почты из раскрывающегося списка. После этого откроется экран редактирования свойств.

На этом экране можно настроить следующие значения свойств:

  • Активирован: Y/N — указывает, следует ли использовать коннектор электронной почты. Значение по умолчанию — «N».
  • Протокол: Тип протокола SMTP.
  • Хост: Абсолютный путь к SMTP-серверу.
  • Порт: Порт SMTP-сервера.
  • Адрес отправителя: Адрес электронной почты отправителя сообщения.
  • Имя пользователя: Имя пользователя для SMTP-сервера.
  • Пароль: Пароль для SMTP-сервера.

Наконец, нажмите кнопку «Проверить соединение», чтобы убедиться в возможности подключения к SMTP-серверу и в корректности настроек.

Редактирование шаблона электронного письма

В окне проекта, в разделе «Конфигурация», затем «Редактировать шаблоны электронных писем», можно изменять шаблоны электронных писем для сообщений.

Вы можете выбрать:

  • Язык шаблона:
  • Тип шаблона:
  • Тема письма:
  • Содержимое шаблона:

Необходимо указать язык, поскольку веб-приложение будет отправлять электронные письма пользователям на том языке, который они выбрали в своих настройках. Необходимо выбрать тип шаблона. Тип соответствует роли пользователя, то есть данное письмо будет отправляться только пользователям с выбранной ролью (типом). Необходимо задать тему письма. Тема — это краткое изложение темы сообщения. Необходимо задать содержимое письма. Это любая информация, которую вы хотите отправить пользователю. В сообщении также можно использовать определённые ключевые слова; веб-приложение обработает их и заменит ключевые слова новыми значениями.

Планирование отправки электронных писем

Отправка электронных писем может выполняться только через планировщик. Перейдите в «Конфигурацию», затем на экран «Планирование заданий». На этом экране можно настроить задание отправки электронных писем. Задание получает список уведомлений по электронной почте, собирает данные и отправляет их на адрес электронной почты пользователя. См. также руководство по планировщику.

Note

Информация об успехе или неудаче будет отображена во всплывающем окне.
