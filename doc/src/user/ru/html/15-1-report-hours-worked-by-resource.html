Назначение

Данный отчёт извлекает список задач и время, которое ресурсы посвятили им за указанный период. Несколько фильтров позволяют пользователям уточнить запрос, чтобы получить только нужную информацию и исключить посторонние данные.

Входные параметры и фильтры

  • Даты.
    • Тип: Необязательный.
    • Два поля дат:
      • Дата начала: Это наиболее ранняя дата для включения рабочих отчётов. Рабочие отчёты с датами ранее Даты начала исключаются. Если этот параметр оставлен пустым, рабочие отчёты не фильтруются по Дате начала.
      • Дата окончания: Это наиболее поздняя дата для включения рабочих отчётов. Рабочие отчёты с датами позднее Даты окончания исключаются. Если этот параметр оставлен пустым, рабочие отчёты не фильтруются по Дате окончания.
  • Фильтр по работникам: * Тип: Необязательный. * Как работает: Можно выбрать одного или нескольких работников для ограничения рабочих отчётов по времени, отслеженному этими конкретными работниками. Для добавления работника в качестве фильтра найдите его в селекторе и нажмите кнопку Добавить. Если этот фильтр оставлен пустым, рабочие отчёты извлекаются независимо от работника.
  • Фильтр по меткам: * Тип: Необязательный. * Как работает: Можно добавить одну или несколько меток в качестве фильтров, найдя их в селекторе и нажав кнопку Добавить. Эти метки используются для выбора задач, включаемых в результаты при подсчёте посвящённых им часов. Этот фильтр можно применять к табелям учёта рабочего времени, задачам, к обоим или ни к одному.
  • Фильтр по критериям: * Тип: Необязательный. * Как работает: Можно выбрать одно или несколько критериев, найдя их в селекторе и нажав кнопку Добавить. Эти критерии используются для выбора ресурсов, удовлетворяющих хотя бы одному из них. В отчёте будет отображено всё время, посвящённое ресурсами, которые соответствуют одному из выбранных критериев.

Вывод

Заголовок

В заголовке отчёта отображаются фильтры, которые были настроены и применены к текущему отчёту.

Нижний колонтитул

В нижнем колонтитуле указывается дата формирования отчёта.

Тело

Тело отчёта состоит из нескольких групп информации.

  • Первый уровень агрегации — по ресурсу. Всё время, посвящённое ресурсом, отображается вместе под заголовком. Каждый ресурс идентифицируется по:

    • Работник: Фамилия, Имя.
    • Машина: Название.

    Строка-итог показывает общее количество часов, отработанных ресурсом.

  • Второй уровень группировки — по дате. Все отчёты от конкретного ресурса за одну и ту же дату отображаются вместе.

    Строка-итог показывает общее количество часов, отработанных ресурсом за эту дату.

  • Последний уровень перечисляет рабочие отчёты работника за этот день. Для каждой строки рабочего отчёта отображается следующая информация:

    • Код задачи: Код задачи, которой приписаны отслеживаемые часы.
    • Название задачи: Название задачи, которой приписаны отслеживаемые часы.
    • Время начала: Необязательное. Время, когда ресурс начал работать над задачей.
    • Время окончания: Необязательное. Время, когда ресурс закончил работать над задачей в указанную дату.
    • Текстовые поля: Необязательное. Если строка рабочего отчёта содержит текстовые поля, заполненные значения отображаются здесь. Формат: <Название текстового поля>:<Значение>
    • Метки: Зависит от того, есть ли в определении модели рабочего отчёта поле метки. Если меток несколько, они отображаются в одном столбце. Формат: <Название типа метки>:<Значение метки>
