Программа управляет двумя различными типами ресурсов: сотрудниками и машинами.

Ресурсы — сотрудники

Ресурсы — сотрудники представляют работников компании. Их ключевые характеристики:

  • Они соответствуют одному или нескольким общим или специфическим для сотрудников критериям.
  • Они могут быть конкретно назначены на задачу.
  • Они могут быть общим образом назначены на задачу, требующую критерия ресурса.
  • У них может быть стандартный или специфический календарь по мере необходимости.

Машинные ресурсы

Машинные ресурсы представляют оборудование компании. Их ключевые характеристики:

  • Они соответствуют одному или нескольким общим или специфическим для машин критериям.
  • Они могут быть конкретно назначены на задачу.
  • Они могут быть общим образом назначены на задачу, требующую машинного критерия.
  • У них может быть стандартный или специфический календарь по мере необходимости.
  • Программа включает экран настройки, где можно задать значение alpha для представления соотношения машина/сотрудник.
    • Значение alpha указывает количество рабочего времени сотрудника, необходимого для управления машиной. Например, значение alpha 0,5 означает, что каждые 8 часов работы машины требуют 4 часов рабочего времени сотрудника.
    • Пользователи могут назначить значение alpha конкретно работнику, указав, что этот работник управляет машиной в течение этого процента времени.
    • Пользователи также могут сделать общее назначение на основе критерия, чтобы процент использования был присвоен всем ресурсам, соответствующим этому критерию и имеющим свободное время. Общее назначение работает аналогично общему назначению для задач, как описано ранее.

Управление ресурсами

Пользователи могут создавать, редактировать и деактивировать (но не удалять навсегда) работников и машины в компании, перейдя в раздел «Ресурсы». Этот раздел предоставляет следующие функции:

  • Список работников: Отображает пронумерованный список работников, позволяя пользователям управлять их данными.
  • Список машин: Отображает пронумерованный список машин, позволяя пользователям управлять их данными.

Управление работниками

Управление работниками осуществляется через раздел «Ресурсы», затем выбор «Список работников». Пользователи могут редактировать любого работника в списке, нажав стандартный значок редактирования.

При редактировании работника пользователи могут получить доступ к следующим вкладкам:

  1. Сведения о работнике: Эта вкладка позволяет пользователям редактировать основные идентификационные данные работника:

    • Имя
    • Фамилия(и)
    • Национальный документ удостоверения личности (DNI)
    • Ресурс, основанный на очереди (см. раздел «Ресурсы на основе очереди»)

    Редактирование персональных данных работников

  2. Критерии: Эта вкладка используется для настройки критериев, которым соответствует работник. Пользователи могут назначить любые работнические или общие критерии, которые они считают подходящими. Очень важно, чтобы работники соответствовали критериям для максимальной функциональности программы. Для назначения критериев:

    1. Нажмите кнопку «Добавить критерии».
    2. Найдите критерий для добавления и выберите наиболее подходящий.
    3. Нажмите кнопку «Добавить».
    4. Выберите дату начала применения критерия.
    5. Выберите дату окончания применения критерия к ресурсу. Эта дата необязательна, если критерий считается бессрочным.

    Связывание критериев с работниками

  3. Календарь: Эта вкладка позволяет пользователям настраивать конкретный календарь для работника. Всем работникам назначается календарь по умолчанию; однако можно назначить конкретный календарь каждому работнику на основе существующего.

    Вкладка «Календарь» для ресурса

  4. Категория затрат: Эта вкладка позволяет пользователям настраивать категорию затрат, которой работник соответствует в течение данного периода. Эта информация используется для расчёта затрат, связанных с работником в проекте.

    Вкладка «Категория затрат» для ресурса

Назначение ресурсов объясняется в разделе «Назначение ресурсов».

Управление машинами

Машины рассматриваются как ресурсы в любых целях. Поэтому, подобно работникам, машины могут управляться и назначаться задачам. Назначение ресурсов рассматривается в разделе «Назначение ресурсов», где будут объяснены специфические функции машин.

Управление машинами осуществляется через пункт меню «Ресурсы». Этот раздел имеет операцию «Список машин», которая отображает машины компании. Пользователи могут редактировать или удалять машину из этого списка.

При редактировании машин система отображает ряд вкладок для управления различными деталями:

  1. Сведения о машине: Эта вкладка позволяет пользователям редактировать идентификационные данные машины:

    1. Название
    2. Код машины
    3. Описание машины

    Редактирование сведений о машине

  2. Критерии: Как и для ресурсов-работников, эта вкладка используется для добавления критериев, которым соответствует машина. Машинам можно назначить два типа критериев: специфические для машин или общие. Критерии работников не могут быть назначены машинам. Для назначения критериев:

    1. Нажмите кнопку «Добавить критерии».
    2. Найдите критерий для добавления и выберите наиболее подходящий.
    3. Выберите дату начала применения критерия.
    4. Выберите дату окончания применения критерия к ресурсу. Эта дата необязательна, если критерий считается бессрочным.
    5. Нажмите кнопку «Сохранить и продолжить».

    Назначение критериев машинам

  3. Календарь: Эта вкладка позволяет пользователям настраивать конкретный календарь для машины. Всем машинам назначается календарь по умолчанию; однако можно назначить конкретный календарь каждой машине на основе существующего.

    Назначение календарей машинам

  4. Конфигурация машины: Эта вкладка позволяет пользователям настраивать соотношение машин к ресурсам-работникам. Машина имеет значение alpha, указывающее соотношение машина/работник. Как упоминалось ранее, значение alpha 0,5 указывает, что для каждого полного дня работы машины требуется 0,5 человека. На основе значения alpha система автоматически назначает работников, связанных с машиной, как только машина назначена на задачу. Связывание работника с машиной может осуществляться двумя способами:

    1. Конкретное назначение: Назначьте диапазон дат, в течение которых работник назначен на машину. Это конкретное назначение, поскольку система автоматически назначает часы работнику при планировании машины.
    2. Общее назначение: Назначьте критерии, которым должны соответствовать работники, назначенные на машину. Это создаёт общее назначение работников, соответствующих критериям.

    Конфигурация машин

  5. Категория затрат: Эта вкладка позволяет пользователям настраивать категорию затрат, которой машина соответствует в течение данного периода. Эта информация используется для расчёта затрат, связанных с машиной в проекте.

    Назначение категорий затрат машинам

Виртуальные группы работников

Программа позволяет пользователям создавать виртуальные группы работников, которые не являются реальными работниками, а смоделированным персоналом. Эти группы позволяют пользователям моделировать увеличение производственных мощностей в определённое время на основе настроек календаря.

Виртуальные группы работников позволяют пользователям оценить, как на планирование проекта повлияет найм и назначение персонала, соответствующего конкретным критериям, тем самым помогая в процессе принятия решений.

Вкладки для создания виртуальных групп работников такие же, как для настройки работников:

  • Общие сведения
  • Назначенные критерии
  • Календари
  • Связанные часы

Разница между виртуальными группами работников и реальными работниками заключается в том, что виртуальные группы работников имеют имя для группы и количество, представляющее число реальных людей в группе. Также есть поле для комментариев, где можно предоставить дополнительную информацию, например, для какого проекта потребуется найм, эквивалентный виртуальной группе работников.

Ресурсы на основе очереди

Ресурсы на основе очереди — это особый тип производительного элемента, который может быть либо не назначен, либо иметь 100% загруженность. Иными словами, у них не может быть запланировано более одной задачи одновременно, и они не могут быть перегружены.

Для каждого ресурса на основе очереди автоматически создаётся очередь. Задачи, запланированные для этих ресурсов, могут управляться конкретным образом с использованием предоставленных методов назначения, создавая автоматические назначения между задачами и очередями, соответствующими необходимым критериям, или путём перемещения задач между очередями.
