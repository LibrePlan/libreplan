Планировщик предназначен для динамического планирования заданий. Он разработан с использованием Spring Framework Quartz scheduler.

Для эффективного использования планировщика сначала необходимо создать задания (задания Quartz), которые должны быть запланированы. Затем эти задания можно добавить в базу данных, поскольку все задания, подлежащие планированию, хранятся в базе данных.

При запуске планировщик считывает из базы данных задания, которые необходимо запланировать или снять с расписания, и выполняет соответствующие действия. После этого задания можно динамически добавлять, обновлять или удалять с помощью пользовательского интерфейса Планирование заданий.

Note

Планировщик запускается при запуске веб-приложения LibrePlan и останавливается при его остановке.

Note

Данный планировщик поддерживает только cron-выражения для планирования заданий.

Критерии, используемые планировщиком для планирования или удаления заданий при запуске, следующие:

Для всех заданий:

  • Запланировать
    • Задание имеет Коннектор, Коннектор активирован, и задание разрешено к планированию.
    • Задание не имеет Коннектора и разрешено к планированию.
  • Удалить
    • Задание имеет Коннектор, и Коннектор не активирован.
    • Задание имеет Коннектор, Коннектор активирован, но задание не разрешено к планированию.
    • Задание не имеет Коннектора и не разрешено к планированию.

Note

Задания не могут быть перепланированы или сняты с расписания, если они в настоящее время выполняются.

Представление списка планирования заданий

Представление Список планирования заданий позволяет пользователям:

  • Добавлять новое задание.
  • Редактировать существующее задание.
  • Удалять задание.
  • Запускать процесс вручную.

Добавление или редактирование задания

В представлении Список планирования заданий нажмите:

  • Создать для добавления нового задания, или
  • Редактировать для изменения выбранного задания.

Оба действия откроют форму создания/редактирования задания. Форма отображает следующие свойства:

  • Поля:
    • Группа заданий: Название группы заданий.
    • Название задания: Название задания.
    • Cron-выражение: Поле только для чтения с кнопкой Редактировать для открытия окна ввода cron-выражения.
    • Имя класса задания: Раскрывающийся список для выбора задания (из существующих заданий).
    • Коннектор: Раскрывающийся список для выбора коннектора. Необязательный параметр.
    • Расписание: Флажок, указывающий, следует ли планировать данное задание.
  • Кнопки:
    • Сохранить: Сохранение или обновление задания в базе данных и планировщике. После этого пользователь возвращается в Представление списка планирования заданий.
    • Сохранить и продолжить: То же, что «Сохранить», но пользователь не возвращается в Представление списка планирования заданий.
    • Отмена: Ничего не сохраняется, пользователь возвращается в Представление списка планирования заданий.
  • Раздел подсказок о синтаксисе cron-выражений.

Всплывающее окно cron-выражения

Для корректного ввода cron-выражения используется всплывающая форма cron-выражения. В этой форме можно ввести нужное cron-выражение. См. также подсказку о cron-выражении. При вводе недопустимого cron-выражения вы будете немедленно уведомлены.

Удаление задания

Нажмите кнопку Удалить, чтобы удалить задание из базы данных и планировщика. Результат этого действия (успех или неудача) будет отображён.

Ручной запуск задания

В качестве альтернативы ожиданию запуска задания по расписанию можно нажать эту кнопку, чтобы запустить процесс непосредственно. После этого информация об успехе или неудаче будет отображена во всплывающем окне.
