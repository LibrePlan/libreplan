Назначение

Данный отчёт предоставляет обзор статуса проектов с учётом как прогресса, так и затрат.

Он анализирует текущий прогресс каждого проекта, сравнивая его с запланированным прогрессом и выполненной работой.

Отчёт также отображает несколько коэффициентов, связанных со стоимостью проекта, сравнивая текущую эффективность с запланированной.

Входные параметры и фильтры

Существует несколько обязательных параметров:

  • Референсная дата: Это дата, используемая в качестве точки отсчёта для сравнения запланированного статуса проекта с его фактическими показателями. Значение по умолчанию для этого поля — текущая дата.
  • Тип прогресса: Это тип прогресса, используемый для измерения хода выполнения проекта. Приложение позволяет одновременно измерять проект с использованием различных типов прогресса. Тип, выбранный пользователем в выпадающем меню, используется для расчёта данных отчёта. Значение по умолчанию для типа прогресса — spread (распределённый), что является специальным типом прогресса, использующим предпочтительный метод измерения прогресса, настроенный для каждого элемента WBS.

Необязательные параметры:

  • Дата начала: Это наиболее ранняя дата начала для проектов, включаемых в отчёт. Если это поле оставлено пустым, минимальной даты начала для проектов нет.
  • Дата окончания: Это наиболее поздняя дата окончания для проектов, включаемых в отчёт. Все проекты, завершающиеся после Даты окончания, будут исключены.
  • Фильтр по проектам: Этот фильтр позволяет пользователям выбирать конкретные проекты для включения в отчёт. Если в фильтр не добавлены проекты, отчёт будет включать все проекты в базе данных. Предоставляется выпадающее меню с поиском для нахождения нужного проекта. Проекты добавляются в фильтр нажатием кнопки Добавить.

Заголовок

В заголовке отчёта отображаются следующие поля:

  • Дата начала: Дата начала фильтрации. Не отображается, если отчёт не фильтруется по этому полю.
  • Дата окончания: Дата окончания фильтрации. Не отображается, если отчёт не фильтруется по этому полю.
  • Тип прогресса: Тип прогресса, использованный в отчёте.
  • Проекты: Указывает отфильтрованные проекты, для которых формируется отчёт. Будет отображена строка Все, когда отчёт включает все проекты, удовлетворяющие другим фильтрам.
  • Референсная дата: Обязательная входная референсная дата, выбранная для отчёта.

Тело

Тело отчёта состоит из списка проектов, выбранных на основе входных фильтров.

Фильтры работают путём добавления условий, за исключением набора, образованного фильтрами по датам (Дата начала, Дата окончания) и Фильтром по проектам. В этом случае, если один или оба фильтра по датам заполнены, а Фильтр по проектам содержит список выбранных проектов, последний фильтр имеет приоритет. Это означает, что проекты, включённые в отчёт, — это проекты, указанные в Фильтре по проектам, независимо от фильтров по датам.

Важно отметить, что прогресс в отчёте рассчитывается как доля единицы, в диапазоне от 0 до 1.

Для каждого проекта, выбранного для включения в вывод отчёта, отображается следующая информация:

  • Название проекта.
  • Общие часы. Общие часы по проекту отображаются путём суммирования часов по каждой задаче. Отображаются два типа общих часов:
    • Оценочные (ОЦ). Это сумма всех оценочных часов в WBS проекта. Она представляет общее количество часов, оцениваемых для завершения проекта.
    • Плановые (ПЛ). В LibrePlan возможно иметь два различных значения: оценочное количество часов для задачи (количество часов, изначально оценённых для выполнения задачи) и плановые часы (часы, выделенные в плане для выполнения задачи). Плановые часы могут быть равны, меньше или больше оценочных и определяются на более позднем этапе — при операции назначения. Поэтому общие плановые часы по проекту — это сумма всех выделенных часов по его задачам.
  • Прогресс. Отображаются три измерения, связанных с общим прогрессом типа, указанного в фильтре входного прогресса для каждого проекта на референсную дату:
    • Измеренный (ИЗМ). Это общий прогресс с учётом измерений прогресса с датой ранее Референсной даты во входных параметрах отчёта. Учитываются все задачи, и сумма взвешивается по количеству часов для каждой задачи.
    • Вменённый (ВМН). Это прогресс при условии, что работа продолжается в том же темпе, что и выполненные часы для задачи. Если выполнено X часов из Y часов для задачи, общий вменённый прогресс считается равным X/Y.
    • Плановый (ПЛН). Это общий прогресс проекта согласно плановому расписанию на референсную дату. Если всё происходило именно так, как запланировано, измеренный прогресс должен совпадать с плановым.
  • Часы до даты. Два поля показывают количество часов до референсной даты с двух точек зрения:
    • Плановые (ЧПЛ). Это сумма часов, выделенных для любой задачи в проекте с датой, меньшей или равной Референсной дате.
    • Фактические (ЧФК). Это сумма часов, отражённых в рабочих отчётах для любой задачи в проекте с датой, меньшей или равной Референсной дате.
  • Разница. В этом разделе представлено несколько показателей, связанных со стоимостью:
    • Стоимость. Это разница в часах между затраченными часами с учётом измеренного прогресса и часами, выполненными до референсной даты. Формула: ИЗМ*ПЛ - ЧФК.
    • Плановая. Это разница между часами, затраченными согласно общему измеренному прогрессу проекта, и числом, запланированным до Референсной даты. Измеряет преимущество или задержку по времени. Формула: ИЗМ*ПЛ - ЧПЛ.
    • Коэффициент стоимости. Рассчитывается путём деления ИЗМ / ВМН. Если больше 1, проект прибыльный на данном этапе. Если меньше 1, проект убыточный.
    • Плановый коэффициент. Рассчитывается путём деления ИЗМ / ПЛН. Если больше 1, проект опережает расписание. Если меньше 1, проект отстаёт от расписания.
