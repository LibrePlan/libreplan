標準
####

.. contents::

標準是程式中用於對資源和任務進行分類的元素。任務需要特定的標準，而資源必須符合這些標準。

以下是標準使用方式的範例：一個資源被分配了「焊接工」標準（意味著該資源符合「焊接工」類別），而一個任務需要「焊接工」標準才能完成。因此，當使用通用分配（而非特定分配）將資源分配給任務時，具有「焊接工」標準的員工將被納入考量。有關不同分配類型的更多資訊，請參閱資源分配章節。

程式允許對標準執行以下操作：

*   標準管理
*   為資源分配標準
*   為任務分配標準
*   根據標準篩選實體。可以按標準篩選任務和專案項目，以在程式中執行各種操作。

本節僅說明第一個功能：標準管理。兩種分配類型將在後面說明：「資源管理」章節中的資源分配，以及「任務規劃」章節中的篩選。

標準管理
========

可透過管理選單存取標準管理：

.. figure:: images/menu.png
   :scale: 50

   一級選單索引標籤

管理標準的具體操作是 *管理標準*。此操作允許您列出系統中可用的標準。

.. figure:: images/lista-criterios.png
   :scale: 50

   標準清單

您可以透過點擊 *建立* 按鈕來存取建立/編輯標準表單。要編輯現有標準，請點擊編輯圖示。

.. figure:: images/edicion-criterio.png
   :scale: 50

   編輯標準

如上圖所示，標準編輯表單允許您執行以下操作：

*   **編輯標準的名稱。**
*   **指定是否可以同時分配多個值，或者所選標準類型只能有一個值。** 例如，一個資源可以同時符合「焊接工」和「車床操作員」兩個標準。
*   **指定標準類型：**

    *   **通用：** 可用於機器和員工的標準。
    *   **員工：** 只能用於員工的標準。
    *   **機器：** 只能用於機器的標準。

*   **指出標準是否為層次結構。** 有時，標準需要以層次結構方式處理。例如，將標準分配給一個元素不會自動將其分配給由其衍生的元素。層次結構標準的一個清晰範例是「位置」。例如，被指定為「加利西亞」位置的人也會屬於「西班牙」。
*   **指出標準是否已授權。** 這是使用者停用標準的方式。一旦標準已建立並用於歷史資料，就無法更改。取而代之，可以將其停用以防止其出現在選擇清單中。
*   **描述標準。**
*   **新增新值。** 表單的第二部分有一個帶有 *新標準* 按鈕的文字輸入欄位。
*   **編輯現有標準值的名稱。**
*   **在目前標準值清單中上下移動標準值。**
*   **從清單中移除標準值。**

標準管理表單遵循簡介中描述的表單行為，提供三種操作：*儲存*、*儲存並關閉* 和 *關閉*。
