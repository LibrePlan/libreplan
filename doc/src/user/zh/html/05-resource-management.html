程式管理兩種不同類型的資源：員工和機器。

員工資源

員工資源代表公司的員工。其主要特性包括：

  • 符合一個或多個通用或員工特定標準。
  • 可以特定分配給任務。
  • 可以通用分配給需要資源標準的任務。
  • 可以根據需要擁有預設或特定的行事曆。

機器資源

機器資源代表公司的機械設備。其主要特性包括：

  • 符合一個或多個通用或機器特定標準。
  • 可以特定分配給任務。
  • 可以通用分配給需要機器標準的任務。
  • 可以根據需要擁有預設或特定的行事曆。
  • 程式包含一個設定畫面，可以定義 alpha 值來表示機器/員工比率。
    • alpha 值表示操作機器所需的員工時間量。例如，alpha 值為 0.5 意味著每 8 小時的機器運作需要 4 小時的員工時間。
    • 使用者可以特定將 alpha 值分配給員工，指定該員工在該比例時間內操作機器。
    • 使用者也可以基於標準進行通用分配，以便將使用百分比分配給符合該標準且有可用時間的所有資源。通用分配的工作方式與之前描述的任務通用分配類似。

管理資源

使用者可以透過導覽至「資源」部分在公司內建立、編輯和停用（但不能永久刪除）員工和機器。此部分提供以下功能：

  • 員工清單： 顯示員工的編號清單，允許使用者管理其詳細資訊。
  • 機器清單： 顯示機器的編號清單，允許使用者管理其詳細資訊。

管理員工

員工管理透過前往「資源」部分，然後選擇「員工清單」來存取。使用者可以透過點擊標準編輯圖示來編輯清單中的任何員工。

編輯員工時，使用者可以存取以下索引標籤：

  1. 員工詳細資訊： 此索引標籤允許使用者編輯員工的基本識別詳細資訊：

    • 名字
    • 姓氏
    • 國家身份證件（DNI）
    • 佇列型資源（見佇列型資源章節）

    編輯員工個人詳細資訊

  2. 標準： 此索引標籤用於設定員工符合的標準。使用者可以分配任何他們認為合適的員工或通用標準。員工符合標準對於最大化程式功能至關重要。要分配標準：

    1. 點擊「新增標準」按鈕。
    2. 搜尋要新增的標準並選擇最合適的一個。
    3. 點擊「新增」按鈕。
    4. 選擇標準開始適用的開始日期。
    5. 選擇將標準套用於資源的結束日期。如果標準被認為是無限期的，此日期為選填。

    將標準與員工相關聯

  3. 行事曆： 此索引標籤允許使用者為員工設定特定的行事曆。所有員工都分配了預設行事曆；但是，可以根據現有行事曆為每個員工分配特定的行事曆。

    資源的行事曆索引標籤

  4. 成本類別： 此索引標籤允許使用者設定員工在給定期間符合的成本類別。此資訊用於計算與員工相關的專案成本。

    資源的成本類別索引標籤

資源分配在「資源分配」部分說明。

管理機器

機器在所有方面都被視為資源。因此，與員工類似，機器可以被管理並分配給任務。資源分配在「資源分配」部分涵蓋，將說明機器的特定功能。

機器從「資源」選單項目進行管理。此部分有一個稱為「機器清單」的操作，顯示公司的機器。使用者可以從此清單編輯或刪除機器。

編輯機器時，系統顯示一系列索引標籤以管理不同的詳細資訊：

  1. 機器詳細資訊： 此索引標籤允許使用者編輯機器的識別詳細資訊：

    1. 名稱
    2. 機器代碼
    3. 機器描述

    編輯機器詳細資訊

  2. 標準： 與員工資源一樣，此索引標籤用於新增機器符合的標準。可以為機器分配兩種類型的標準：機器特定或通用。員工標準不能分配給機器。要分配標準：

    1. 點擊「新增標準」按鈕。
    2. 搜尋要新增的標準並選擇最合適的一個。
    3. 選擇標準開始適用的開始日期。
    4. 選擇將標準套用於資源的結束日期。如果標準被認為是無限期的，此日期為選填。
    5. 點擊「儲存並繼續」按鈕。

    為機器分配標準

  3. 行事曆： 此索引標籤允許使用者為機器設定特定的行事曆。所有機器都分配了預設行事曆；但是，可以根據現有行事曆為每台機器分配特定的行事曆。

    為機器分配行事曆

  4. 機器設定： 此索引標籤允許使用者設定機器與員工資源的比率。機器有一個 alpha 值，表示機器/員工比率。如前所述，alpha 值為 0.5 表示每個完整機器運作日需要 0.5 個員工。根據 alpha 值，一旦機器被分配給任務，系統會自動分配與機器相關聯的員工。將員工與機器相關聯可以透過兩種方式完成：

    1. 特定分配： 指定員工被分配給機器的日期範圍。這是特定分配，因為當機器被排程時，系統會自動為員工分配工時。
    2. 通用分配： 指定必須由分配給機器的員工符合的標準。這建立了符合標準的員工的通用分配。

    機器設定

  5. 成本類別： 此索引標籤允許使用者設定機器在給定期間符合的成本類別。此資訊用於計算與機器相關的專案成本。

    為機器分配成本類別

虛擬員工群組

程式允許使用者建立虛擬員工群組，這些不是真實員工，而是模擬人員。這些群組使使用者能夠根據行事曆設定，模擬特定時間內生產能力增加的情況。

虛擬員工群組允許使用者評估僱用和分配符合特定標準的人員將如何影響專案規劃，從而幫助決策過程。

建立虛擬員工群組的索引標籤與設定員工的索引標籤相同：

  • 一般詳細資訊
  • 分配的標準
  • 行事曆
  • 相關工時

虛擬員工群組和實際員工之間的區別在於虛擬員工群組有一個群組名稱和數量，代表群組中真實人員的數量。還有一個注釋欄位，可以提供額外資訊，例如哪個專案需要相當於虛擬員工群組的招聘。

佇列型資源

佇列型資源是一種特定類型的生產要素，可以是未分配的或 100% 投入的。換句話說，它們不能在同一時間排程多於一個任務，也不能被過度分配。

對於每個佇列型資源，會自動建立一個佇列。為這些資源排程的任務可以使用提供的分配方法進行特定管理，在任務和符合所需標準的佇列之間建立自動分配，或者在佇列之間移動任務。
