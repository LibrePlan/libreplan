專案儀表板是 LibrePlan 的一個視角，包含一組**關鍵績效指標（KPI）**，可協助評估專案在以下方面的表現：

  • 工作進度
  • 成本
  • 已分配資源的狀態
  • 時間限制

進度績效指標

計算兩個指標：專案進度百分比和任務狀態。

專案進度百分比

此圖表顯示專案的整體進度，並與基於 Gantt 圖的預期進度進行比較。

進度由兩個長條表示：

  • 當前進度： 根據已採取的測量值得出的當前進度。
  • 預期進度： 根據專案計劃，專案此時應達到的進度。

若要查看每個長條的實際測量值，請將滑鼠游標懸停在長條上。

整體專案進度使用幾種不同的方法估算，因為沒有單一、普遍正確的方法：

  • 擴散進度： 這是在專案層級設定為擴散進度的進度類型。在此情況下，無法計算預期值，僅顯示當前長條。
  • 依所有任務工時： 所有專案任務的進度取平均值以計算整體值。這是考量分配給每個任務工時數的加權平均值。
  • 依關鍵路徑工時： 屬於任何專案關鍵路徑的任務進度取平均值以獲得整體值。這是考量每個相關任務總分配工時的加權平均值。
  • 依關鍵路徑持續時間： 屬於任何關鍵路徑的任務進度使用加權平均值取平均，但這次考量的是每個相關任務的持續時間而非分配的工時。

任務狀態

圓餅圖顯示不同狀態的專案任務百分比。定義的狀態為：

  • 已完成： 以100%進度值識別的已完成任務。
  • 進行中： 目前正在進行的任務。這些任務的進度值不為0%或100%，或已追蹤了一些工作時間。
  • 準備開始： 進度為0%、無追蹤工時、所有*結束到開始*依賴任務均已*完成*，且所有*開始到開始*依賴任務均已*完成*或*進行中*的任務。
  • 已封鎖： 進度為0%、無追蹤工時，且前置依賴任務既不*進行中*也不在*準備開始*狀態的任務。

成本指標

計算幾個*實獲值管理*成本指標：

  • CV（成本差異）： 目前時刻*實獲值曲線*與*實際成本曲線*之間的差異。正值表示收益，負值表示損失。

  • ACWP（已執行工作的實際成本）： 目前時刻專案中已追蹤的總工時數。

  • CPI（成本績效指數）： *實獲值/實際成本*比率。

    • > 100 為有利，表示專案低於預算。
    • = 100 也為有利，表示成本完全符合計劃。

  • ETC（完工估算）： 完成專案所剩餘的時間。

    • < 0 表示專案超出預算。
    • > 0 表示專案低於預算。

資源

若要從資源角度分析專案，提供兩個比率和一個直方圖。

已完成任務估算偏差直方圖

此直方圖計算分配給專案任務的工時數與實際投入工時數之間的偏差。

偏差以百分比計算所有已完成任務，並在直方圖中表示計算出的偏差。縱軸顯示每個偏差區間內的任務數量。動態計算六個偏差區間。

加班比率

此比率彙總分配給專案任務的資源超載情況。使用公式計算：加班比率 = 超載 / (負載 + 超載)。

  • = 0 為有利，表示資源未超載。
  • > 0 為不利，表示資源超載。

可用性比率

此比率彙總目前分配給專案的資源的空閒容量。因此，它衡量資源在不超載的情況下接受更多分配的可用性。計算公式為：可用性比率 = (1 - 負載/容量) * 100

  • 可能的值介於0%（完全分配）到100%（未分配）之間。

時間

包含兩個圖表：專案任務完成時間偏差的直方圖和截止日期違反的圓餅圖。

任務完成提前或延後

此計算確定專案任務的計劃結束時間與實際結束時間之間的天數差異。計劃完成日期取自 Gantt 圖，實際完成日期取自任務的最後一次追蹤工時。

任務完成的延後或提前以直方圖表示。縱軸顯示提前/延後天數差值對應橫坐標天數區間的任務數量。計算六個動態任務完成偏差區間。

  • 負值表示提前完成。
  • 正值表示延後完成。

截止日期違反

本節計算與專案截止日期（若已設定）的差距。此外，圓餅圖顯示符合截止日期的任務百分比。圖表中包含三種類型的值：

  • 未設定截止日期的任務百分比。
  • 實際結束日期晚於其截止日期的已結束任務百分比。實際結束日期取自任務的最後一次追蹤工時。
  • 實際結束日期早於其截止日期的已結束任務百分比。
