連接器是 LibrePlan 的客戶端應用程式，可用於與（網路）伺服器通訊以擷取資料、處理資料並加以儲存。目前有三種連接器：JIRA 連接器、Tim Enterprise 連接器和電子郵件連接器。

設定

連接器在使用前必須正確設定。可從「主要設定」畫面的「連接器」索引標籤進行設定。

連接器畫面包含：

  • 下拉式清單： 可用連接器的清單。
  • 屬性編輯畫面： 所選連接器的屬性編輯表單。
  • 測試連線按鈕： 用於測試與連接器連線的按鈕。

從連接器的下拉式清單中選擇要設定的連接器，畫面將顯示所選連接器的屬性編輯器表單。在屬性編輯器表單中，您可以視需要變更屬性值，並使用「測試連線」按鈕測試您的設定。

Note

屬性已設有預設值。最重要的屬性是「已啟用」。預設值為「N」，表示除非您將值變更為「Y」並儲存變更，否則不會使用該連接器。

JIRA 連接器

JIRA 是一套問題與專案追蹤系統。

JIRA 連接器是一個應用程式，可用於向 JIRA 網路伺服器請求 JIRA 議題資料並處理回應。請求以 JIRA 標籤為基礎。在 JIRA 中，標籤可用於對議題進行分類。請求的結構如下：擷取所有以此標籤名稱分類的議題。

連接器接收回應（即議題），並將其轉換為 LibrePlan 的「專案元素」和「工時表」。

*JIRA 連接器*在使用前必須正確設定。

設定

從「主要設定」畫面選擇「連接器」索引標籤。在連接器畫面中，從下拉式清單選擇 JIRA 連接器，隨即顯示屬性編輯器畫面。

在此畫面中，您可以設定以下屬性值：

  • 已啟用： Y/N，表示是否要使用 JIRA 連接器。預設值為「N」。
  • 伺服器 URL： JIRA 網路伺服器的絕對路徑。
  • 使用者名稱和密碼： 授權用的使用者憑證。
  • JIRA 標籤：逗號分隔的標籤清單或 URL： 您可以輸入標籤 URL 或逗號分隔的標籤清單。
  • 工時類型： 工作工時的類型。預設值為「預設」。

Note

JIRA 標籤： 目前 JIRA 網路伺服器不支援提供所有可用標籤的清單。作為因應措施，我們開發了一個簡單的 PHP 腳本，在 JIRA 資料庫中執行簡單的 SQL 查詢以擷取所有不重複的標籤。您可以將此 PHP 腳本作為「JIRA 標籤 URL」使用，或在「JIRA 標籤」欄位中以逗號分隔文字輸入所需的標籤。

最後，點擊「測試連線」按鈕，測試是否可以連線至 JIRA 網路伺服器，以及您的設定是否正確。

同步

在專案視窗的「一般資料」下，您可以開始將專案元素與 JIRA 議題進行同步。

點擊「與 JIRA 同步」按鈕以開始同步。

  • 若這是第一次，將顯示一個彈出視窗（含自動完成的標籤清單）。在此視窗中，您可以選擇要同步的標籤，並點擊「開始同步」按鈕開始同步程序，或點擊「取消」按鈕取消。
  • 若標籤已同步，JIRA 畫面將顯示上次同步日期和標籤。在此情況下，不會顯示選擇標籤的彈出視窗，而是直接對該已顯示（已同步）的標籤開始同步程序。

Note

「專案」與「標籤」之間的關係是一對一。一個「專案」只能與一個標籤同步。

Note

成功（重新）同步後，資訊將寫入資料庫，JIRA 畫面將更新為最新的同步日期和標籤。

（重新）同步分兩個階段執行：

  • 第一階段： 同步專案元素，包括進度指派和量測值。
  • 第二階段： 同步工時表。

Note

若第一階段失敗，將不會執行第二階段，也不會將任何資訊寫入資料庫。

Note

成功或失敗的資訊將顯示在彈出視窗中。

同步成功完成後，結果將顯示在「專案詳情」畫面的「工作分解結構（WBS 任務）」索引標籤中。此使用者介面與標準 WBS 相比有兩項變更：

  • 「任務總工時」欄位不可修改（唯讀），因為同步是單向的。任務工時只能在 JIRA 網路伺服器中更新。
  • 「代碼」欄位顯示 JIRA 議題鍵值，同時也是 JIRA 議題的超連結。若要前往該鍵值（JIRA 議題）的文件，請點擊所需的鍵值。

排程

JIRA 議題的重新同步也可以透過排程器執行。前往「作業排程」畫面，在該畫面中可設定 JIRA 作業以執行同步。作業會搜尋資料庫中最後同步的標籤，並依此重新同步。另請參閱排程器手冊。

Tim Enterprise 連接器

Tim Enterprise 是 Aenova 的荷蘭產品，是一款基於網路的應用程式，用於管理專案和任務上所花費的時間。

Tim 連接器是一個可與 Tim Enterprise 伺服器通訊的應用程式，用於：

  • 匯出工作人員（使用者）在專案中所花費的所有工時，這些工時可以在 Tim Enterprise 中登錄。
  • 匯入工作人員（使用者）的所有班表，以有效規劃資源。

*Tim 連接器*在使用前必須正確設定。

設定

從「主要設定」畫面選擇「連接器」索引標籤。在連接器畫面中，從下拉式清單選擇 Tim 連接器，隨即顯示屬性編輯器畫面。

在此畫面中，您可以設定以下屬性值：

  • 已啟用： Y/N，表示是否要使用 Tim 連接器。預設值為「N」。
  • 伺服器 URL： Tim Enterprise 伺服器的絕對路徑。
  • 使用者名稱和密碼： 授權用的使用者憑證。
  • 匯出至 Tim 的工時表天數： 您要匯出工時表的過去天數。
  • 從 Tim 匯入班表的天數： 您要匯入班表的未來天數。
  • 生產力係數： 有效工作時數的百分比。預設值為「100%」。
  • 要匯入班表的部門 ID： 逗號分隔的部門 ID。

最後，點擊「測試連線」按鈕，測試是否可以連線至 Tim Enterprise 伺服器，以及您的設定是否正確。

匯出

在專案視窗的「一般資料」下，您可以開始將工時表匯出至 Tim Enterprise 伺服器。

輸入「Tim 產品代碼」，然後點擊「匯出至 Tim」按鈕以開始匯出。

Tim 連接器會連同產品代碼一起新增以下欄位：

  • 工作人員/使用者的全名。
  • 工作人員執行任務的日期。
  • 投入的工作量或在任務上所花費的工時。
  • 一個選項，指示 Tim Enterprise 是否應更新登錄或插入新的登錄。

Tim Enterprise 的回應只包含記錄 ID（整數）的清單。由於回應清單只包含與請求欄位無關的數字，這使得判斷發生了什麼問題變得困難。若清單中所有條目均不包含「0」值，則視為匯出請求（在 Tim 中登錄）成功。否則，包含「0」值的條目的匯出請求即為失敗。因此，您無法看到哪個請求失敗，因為清單條目只包含值「0」。判斷此問題的唯一方法是檢查 Tim Enterprise 伺服器上的日誌檔案。

Note

成功匯出後，資訊將寫入資料庫，Tim 畫面將更新為最新的匯出日期和產品代碼。

Note

成功或失敗的資訊將顯示在彈出視窗中。

排程匯出

匯出程序也可以透過排程器執行。前往「作業排程」畫面，在該畫面中可設定 Tim 匯出作業。作業會搜尋資料庫中最後匯出的工時表，並依此重新匯出。另請參閱排程器手冊。

匯入

匯入班表只能透過排程器運作。由於使用者無需提供任何輸入，因此沒有為此設計使用者介面。前往「作業排程」畫面，設定 Tim 匯入作業。作業會遍歷連接器屬性中設定的所有部門，並匯入每個部門的所有班表。另請參閱排程器手冊。

匯入時，Tim 連接器在請求中新增以下欄位：

  • 期間： 您要匯入班表的期間（起始日期－截止日期）。這可作為篩選條件提供。
  • 部門： 您要匯入班表的部門。部門可設定。
  • 您感興趣的欄位（如人員資訊、班表類別等），Tim 伺服器應在其回應中包含這些欄位。

匯入回應包含以下欄位，足以管理 LibrePlan 中的例外日期：

  • 人員資訊： 姓名和網路名稱。
  • 部門： 工作人員所在的部門。
  • 班表類別： 工作人員出勤/缺勤（Aanwzig/afwezig）的資訊，以及工作人員缺勤時的原因（LibrePlan 例外類型）。
  • 日期： 工作人員出勤/缺勤的日期。
  • 時間： 出勤/缺勤的開始時間，例如 08:00。
  • 時長： 工作人員出勤/缺勤的小時數。

將匯入回應轉換為 LibrePlan 的「例外日期」時，需考量以下翻譯：

  • 若班表類別包含名稱「Vakantie」，將翻譯為「RESOURCE HOLIDAY」（資源假日）。
  • 班表類別「Feestdag」將翻譯為「BANK HOLIDAY」（法定假日）。
  • 其他所有類別，如「Jus uren」、「PLB uren」等，應手動新增至「行事曆例外日期」。

此外，在匯入回應中，班表每天分為兩或三個部分：例如早班、下午班和晚班。然而，LibrePlan 每天只允許一種「例外類型」。Tim 連接器負責將這些部分合併為一種例外類型。即以時長最長的班表類別作為有效的例外類型，但總時長為這些類別部分的所有時長之和。

與 LibrePlan 不同，在 Tim Enterprise 中，工作人員請假時的總時長表示工作人員在該總時長內不可用。然而，在 LibrePlan 中，若工作人員請假，總時長應為零。Tim 連接器也負責處理此翻譯。

電子郵件連接器

電子郵件是一種從寄件者向一位或多位收件者交換數位訊息的方法。

電子郵件連接器可用於設定簡單郵件傳輸協定（SMTP）伺服器的連線屬性。

*電子郵件連接器*在使用前必須正確設定。

設定

從「主要設定」畫面選擇「連接器」索引標籤。在連接器畫面中，從下拉式清單選擇電子郵件連接器，隨即顯示屬性編輯器畫面。

在此畫面中，您可以設定以下屬性值：

  • 已啟用： Y/N，表示是否要使用電子郵件連接器。預設值為「N」。
  • 協定： SMTP 協定的類型。
  • 主機： SMTP 伺服器的絕對路徑。
  • 連接埠： SMTP 伺服器的連接埠。
  • 寄件地址： 訊息寄件者的電子郵件地址。
  • 使用者名稱： SMTP 伺服器的使用者名稱。
  • 密碼： SMTP 伺服器的密碼。

最後，點擊「測試連線」按鈕，測試是否可以連線至 SMTP 伺服器，以及您的設定是否正確。

編輯電子郵件範本

在專案視窗的「設定」下，然後選擇「編輯電子郵件範本」，您可以修改訊息的電子郵件範本。

您可以選擇：

  • 範本語言：
  • 範本類型：
  • 電子郵件主旨：
  • 範本內容：

您需要指定語言，因為網路應用程式將以使用者在其偏好設定中所選擇的語言傳送電子郵件。您需要選擇範本類型。類型是使用者角色，表示此電子郵件只會傳送給具有所選角色（類型）的使用者。您需要設定電子郵件主旨。主旨是訊息主題的簡短摘要。您需要設定電子郵件內容。這是您要傳送給使用者的任何資訊。訊息中還有一些可使用的關鍵字，網路應用程式將解析這些關鍵字並以新值取代。

排程電子郵件

傳送電子郵件只能透過排程器執行。前往「設定」，然後選擇「作業排程」畫面。在該畫面中，您可以設定電子郵件傳送作業。作業會取得電子郵件通知清單，收集資料，並將其傳送至使用者的電子郵件。另請參閱排程器手冊。

Note

成功或失敗的資訊將顯示在彈出視窗中。
