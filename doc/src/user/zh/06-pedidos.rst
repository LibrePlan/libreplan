專案與專案元素
##############

.. contents::

專案代表程式使用者要執行的工作。每個專案對應公司將提供給其客戶的一個專案。

專案由一個或多個專案元素組成。每個專案元素代表要完成的工作的特定部分，並定義專案上的工作應如何規劃和執行。專案元素按層次結構組織，對層次結構的深度沒有限制。這種層次結構允許繼承某些功能，例如標籤。

以下各節描述使用者可以對專案和專案元素執行的操作。

專案
====

專案代表客戶向公司請求的專案或工作。專案在公司的規劃中識別專案。與全面管理程式不同，LibrePlan 只需要專案的某些關鍵詳細資訊。這些詳細資訊包括：

*   **專案名稱：** 專案的名稱。
*   **專案代碼：** 專案的唯一代碼。
*   **專案總金額：** 專案的總財務價值。
*   **估計開始日期：** 專案的計劃開始日期。
*   **結束日期：** 專案的計劃完成日期。
*   **負責人：** 負責專案的個人。
*   **描述：** 專案的描述。
*   **分配的行事曆：** 與專案相關聯的行事曆。
*   **自動產生代碼：** 指示系統自動為專案元素和工時群組產生代碼的設定。
*   **相依性和限制之間的優先順序：** 使用者可以選擇在發生衝突時相依性或限制哪個優先。

但是，完整的專案還包括其他相關實體：

*   **分配給專案的工時：** 分配給專案的總工時。
*   **歸因於專案的進度：** 專案上取得的進度。
*   **標籤：** 分配給專案的標籤。
*   **分配給專案的標準：** 與專案相關聯的標準。
*   **材料：** 專案所需的材料。
*   **品質表單：** 與專案相關聯的品質表單。

建立或編輯專案可以從程式中的多個位置完成：

*   **從公司概覽中的「專案清單」：**

    *   **編輯：** 在所需專案上點擊編輯按鈕。
    *   **建立：** 點擊「新專案」。

*   **從甘特圖中的專案：** 切換至專案詳細資訊檢視。

編輯專案時，使用者可以存取以下索引標籤：

*   **編輯專案詳細資訊：** 此畫面允許使用者編輯基本專案詳細資訊：

    *   名稱
    *   代碼
    *   估計開始日期
    *   結束日期
    *   負責人
    *   客戶
    *   描述

    .. figure:: images/order-edition.png
       :scale: 50

       編輯專案

*   **專案元素清單：** 此畫面允許使用者對專案元素執行多項操作：

    *   建立新的專案元素。
    *   在層次結構中將專案元素提升一個層級。
    *   在層次結構中將專案元素降低一個層級。
    *   縮排專案元素（在層次結構中向下移動）。
    *   取消縮排專案元素（在層次結構中向上移動）。
    *   篩選專案元素。
    *   刪除專案元素。
    *   透過拖放在層次結構中移動元素。

    .. figure:: images/order-elements-list.png
       :scale: 40

       專案元素清單

*   **分配的工時：** 此畫面顯示歸因於專案的總工時，將專案元素中輸入的工時進行分組。

    .. figure:: images/order-assigned-hours.png
       :scale: 50

       按員工分配歸因於專案的工時

*   **進度：** 此畫面允許使用者為專案分配進度類型並輸入進度測量值。請參閱「進度」部分以了解更多詳情。

*   **標籤：** 此畫面允許使用者為專案分配標籤，並查看之前分配的直接和間接標籤。請參閱以下關於編輯專案元素的部分，以了解標籤管理的詳細說明。

    .. figure:: images/order-labels.png
       :scale: 35

       專案標籤

*   **標準：** 此畫面允許使用者分配將適用於專案內所有任務的標準。這些標準將自動套用於所有專案元素，但已明確無效的那些元素除外。專案元素的工時群組（按標準分組）也可以查看，讓使用者能夠識別專案所需的標準。

    .. figure:: images/order-criterions.png
       :scale: 50

       專案標準

*   **材料：** 此畫面允許使用者為專案分配材料。材料可以從程式中的可用材料類別中選擇。材料的管理方式如下：

    *   選擇畫面底部的「搜尋材料」索引標籤。
    *   輸入文字以搜尋材料，或選擇您要查找材料的類別。
    *   系統篩選結果。
    *   選擇所需材料（可以按住「Ctrl」鍵選擇多個材料）。
    *   點擊「指定」。
    *   系統顯示已分配給專案的材料清單。
    *   選擇要分配給專案的單位和狀態。
    *   點擊「儲存」或「儲存並繼續」。
    *   要管理材料的收貨情況，請點擊「分割」以更改部分材料數量的狀態。

    .. figure:: images/order-material.png
       :scale: 50

       與專案相關聯的材料

*   **品質：** 使用者可以為專案分配品質表單。然後完成此表單以確保執行與專案相關聯的某些活動。請參閱以下關於編輯專案元素的部分，以了解管理品質表單的詳情。

    .. figure:: images/order-quality.png
       :scale: 50

       與專案相關聯的品質表單

編輯專案元素
============

專案元素透過點擊編輯圖示從「專案元素清單」索引標籤進行編輯。這將開啟一個新畫面，使用者可以在其中：

*   編輯專案元素的資訊。
*   查看歸因於專案元素的工時。
*   管理專案元素的進度。
*   管理專案標籤。
*   管理專案元素所需的標準。
*   管理材料。
*   管理品質表單。

以下各節詳細描述這些操作中的每一個。

編輯專案元素的資訊
------------------

編輯專案元素的資訊包括修改以下詳細資訊：

*   **專案元素名稱：** 專案元素的名稱。
*   **專案元素代碼：** 專案元素的唯一代碼。
*   **開始日期：** 專案元素的計劃開始日期。
*   **估計結束日期：** 專案元素的計劃完成日期。
*   **總工時：** 分配給專案元素的總工時。這些工時可以從新增的工時群組計算，也可以直接輸入。如果直接輸入，工時必須在工時群組之間分配，如果百分比與初始百分比不匹配，則需要建立新的工時群組。
*   **工時群組：** 可以向專案元素新增一個或多個工時群組。**這些工時群組的目的**是定義將被分配執行工作的資源要求。
*   **標準：** 可以新增必須滿足的標準，以啟用專案元素的通用分配。

.. figure:: images/order-element-edition.png
   :scale: 50

   編輯專案元素

查看歸因於專案元素的工時
------------------------

「分配工時」索引標籤允許使用者查看與專案元素相關聯的工作報告，並查看估計工時中已完成的數量。

.. figure:: images/order-element-hours.png
   :scale: 50

   分配給專案元素的工時

畫面分為兩部分：

*   **工作報告清單：** 使用者可以查看與專案元素相關聯的工作報告清單，包括日期和時間、資源以及用於任務的工時數。
*   **估計工時的使用情況：** 系統計算用於任務的總工時，並將其與估計工時進行比較。

管理專案元素的進度
------------------

輸入進度類型和管理專案元素進度在「進度」章節中描述。

管理專案標籤
------------

如標籤章節所述，標籤允許使用者對專案元素進行分類。這使使用者能夠根據這些標籤對規劃或專案資訊進行分組。

使用者可以直接將標籤分配給專案元素，或分配給層次結構中較高層級的專案元素。使用任一方法分配標籤後，專案元素和相關規劃任務都會與標籤相關聯，並可用於後續篩選。

.. figure:: images/order-element-tags.png
   :scale: 50

   為專案元素分配標籤

如圖所示，使用者可以從 **標籤** 索引標籤執行以下操作：

*   **查看繼承的標籤：** 查看與專案元素相關聯的標籤，這些標籤是從較高層級的專案元素繼承的。與每個專案元素相關聯的規劃任務具有相同的相關標籤。
*   **查看直接分配的標籤：** 查看使用較低層級標籤的分配表單直接與專案元素相關聯的標籤。
*   **分配現有標籤：** 在直接標籤清單下方的表單中搜尋可用標籤來分配標籤。要搜尋標籤，請點擊放大鏡圖示，或在文字方塊中輸入標籤的前幾個字母以顯示可用選項。
*   **建立並分配新標籤：** 從此表單建立與現有標籤類型相關聯的新標籤。為此，選擇一個標籤類型並輸入所選類型的標籤值。點擊「建立並分配」時，系統自動建立標籤並將其分配給專案元素。

管理專案元素和工時群組所需的標準
--------------------------------

專案和專案元素都可以分配必須滿足才能執行工作的標準。標準可以是直接的或間接的：

*   **直接標準：** 這些是直接分配給專案元素的標準。它們是專案元素上的工時群組所需的標準。
*   **間接標準：** 這些是分配給層次結構中較高層級專案元素的標準，並由正在編輯的元素繼承。

除了所需標準外，還可以定義作為專案元素一部分的一個或多個工時群組。這取決於專案元素是否包含其他專案元素作為子節點，或者它是否是葉節點。在第一種情況下，只能查看工時和工時群組的資訊。但是，葉節點可以編輯。葉節點的工作方式如下：

*   系統建立一個與專案元素相關聯的預設工時群組。工時群組可以修改的詳細資訊包括：

    *   **代碼：** 工時群組的代碼（如果不是自動產生的）。
    *   **標準類型：** 使用者可以選擇分配機器或員工標準。
    *   **工時數：** 工時群組中的工時數。
    *   **標準清單：** 要套用於工時群組的標準。要新增新標準，請點擊「新增標準」並從點擊按鈕後出現的搜尋引擎中選擇一個。

*   使用者可以新增具有與之前工時群組不同特性的新工時群組。例如，一個專案元素可能需要焊接工（30 小時）和油漆工（40 小時）。

.. figure:: images/order-element-criterion.png
   :scale: 50

   為專案元素分配標準

管理材料
--------

材料在專案中以與每個專案元素或專案相關聯的清單進行管理。材料清單包含以下欄位：

*   **代碼：** 材料代碼。
*   **日期：** 與材料相關聯的日期。
*   **單位：** 所需單位數。
*   **單位類型：** 用於衡量材料的單位類型。
*   **單位價格：** 每單位價格。
*   **總價格：** 總價格（透過將單位價格乘以單位數計算）。
*   **類別：** 材料所屬的類別。
*   **狀態：** 材料的狀態（例如，已收到、已請求、待處理、處理中、已取消）。

使用材料的步驟如下：

*   在專案元素上選擇「材料」索引標籤。
*   系統顯示兩個子索引標籤：「材料」和「搜尋材料」。
*   如果專案元素沒有分配材料，第一個索引標籤將為空。
*   點擊視窗左下部分的「搜尋材料」。
*   系統顯示可用類別和相關材料的清單。

.. figure:: images/order-element-material-search.png
   :scale: 50

   搜尋材料

*   選擇類別以精細化材料搜尋。
*   系統顯示屬於所選類別的材料。
*   從材料清單中，選擇要分配給專案元素的材料。
*   點擊「指定」。
*   系統在「材料」索引標籤上顯示選定的材料清單，並提供新的欄位供完成。

.. figure:: images/order-element-material-assign.png
   :scale: 50

   為專案元素分配材料

*   選擇分配材料的單位、狀態和日期。

對於後續的材料監控，可以更改一組已收到材料單位的狀態。操作如下：

*   點擊材料清單中每行右側的「分割」按鈕。
*   選擇將行分割成的單位數。
*   程式顯示兩行，材料已被分割。
*   更改包含材料的行的狀態。

使用此分割工具的優點是能夠接收材料的部分交貨，而無需等待整個交貨才能將其標記為已收到。

管理品質表單
------------

某些專案元素需要認證某些任務已完成，才能將其標記為完成。這就是程式具有品質表單的原因，品質表單由一份問題清單組成，如果答案為肯定則認為這些問題很重要。

重要的是要注意，品質表單必須事先建立才能分配給專案元素。

要管理品質表單：

*   前往「品質表單」索引標籤。

    .. figure:: images/order-element-quality.png
       :scale: 50

       為專案元素分配品質表單

*   程式有一個品質表單搜尋引擎。有兩種類型的品質表單：按元素或按百分比。

    *   **元素：** 每個元素是獨立的。
    *   **百分比：** 每個問題將專案元素的進度增加一個百分比。這些百分比必須能夠加起來達到 100%。

*   選擇在管理介面中建立的其中一個表單，然後點擊「指定」。
*   程式從分配給專案元素的表單清單中分配所選表單。
*   點擊專案元素上的「編輯」按鈕。
*   程式在下方清單中顯示品質表單的問題。
*   將已完成的問題標記為已達成。

    *   如果品質表單基於百分比，則問題按順序回答。
    *   如果品質表單基於元素，則問題可以以任何順序回答。
