Управління ресурсами
####################

.. _recursos:
.. contents::

Програма управляє двома окремими типами ресурсів: персоналом і машинами.

Ресурси персоналу
-----------------

Ресурси персоналу представляють працівників компанії. їхні ключові характеристики:

*   Вони відповідають одному або кільком загальним або специфічним для працівників критеріям.
*   Їх можна конкретно призначити до завдання.
*   Їх можна загально призначити до завдання, що вимагає критерію ресурсу.
*   Вони можуть мати стандартний або специфічний календар залежно від потреб.

Ресурси машин
--------------

Ресурси машин представляють обладнання компанії. Їхні ключові характеристики:

*   Вони відповідають одному або кільком загальним або специфічним для машин критеріям.
*   Їх можна конкретно призначити до завдання.
*   Їх можна загально призначити до завдання, що вимагає критерію машини.
*   Вони можуть мати стандартний або специфічний календар залежно від потреб.
*   Програма включає екран конфігурації, де можна визначити значення *альфа* для позначення відношення машина/працівник.

    *   Значення *альфа* вказує на кількість робочого часу, необхідного для керування машиною. Наприклад, значення альфа 0,5 означає, що кожні 8 годин роботи машини вимагають 4 години часу працівника.
    *   Користувачі можуть призначити значення *альфа* конкретно для працівника, призначаючи цього працівника для керування машиною протягом такого відсотку часу.
    *   Користувачі також можуть робити загальне призначення на основі критерію, так що відсоток використання призначається всім ресурсам, які відповідають цьому критерію та мають вільний час. Загальне призначення працює аналогічно до загального призначення для завдань, як описано раніше.

Управління ресурсами
---------------------

Користувачі можуть створювати, редагувати та деактивувати (але не видаляти назавжди) працівників і машини в компанії, перейшовши до розділу «Ресурси». Цей розділ надає такі функції:

*   **Список працівників:** Відображає пронумерований список працівників, що дозволяє користувачам управляти їхніми даними.
*   **Список машин:** Відображає пронумерований список машин, що дозволяє користувачам управляти їхніми даними.

Управління працівниками
========================

Управління працівниками здійснюється через перехід до розділу «Ресурси» та вибір «Список працівників». Користувачі можуть редагувати будь-якого працівника зі списку, натискаючи стандартний значок редагування.

При редагуванні працівника користувачі можуть отримати доступ до таких вкладок:

1.  **Дані працівника:** Ця вкладка дозволяє редагувати основні ідентифікаційні дані працівника:

    *   Ім'я
    *   Прізвище(-а)
    *   Ідентифікаційний документ (ІД)
    *   Ресурс на основі черги (дивіться розділ про ресурси на основі черги)

    .. figure:: images/worker-personal-data.png
       :scale: 50

       Редагування особистих даних працівників

2.  **Критерії:** Ця вкладка використовується для налаштування критеріїв, яким відповідає працівник. Користувачі можуть призначити будь-які критерії для працівників або загальні критерії, які вважають відповідними. Для максимальної функціональності програми вкрай важливо, щоб працівники відповідали критеріям. Щоб призначити критерії:

    i.  Натисніть кнопку «Додати критерії».
    ii. Знайдіть критерій для додавання та виберіть найбільш відповідний.
    iii. Натисніть кнопку «Додати».
    iv. Виберіть дату початку застосування критерію.
    v.  Виберіть дату закінчення застосування критерію до ресурсу. Ця дата є необов'язковою, якщо критерій вважається безстроковим.

    .. figure:: images/worker-criterions.png
       :scale: 50

       Пов'язування критеріїв із працівниками

3.  **Календар:** Ця вкладка дозволяє користувачам налаштовувати специфічний календар для працівника. Усім працівникам призначено стандартний календар; однак можна призначити специфічний календар кожному працівнику на основі наявного.

    .. figure:: images/worker-calendar.png
       :scale: 50

       Вкладка календаря для ресурсу

4.  **Категорія витрат:** Ця вкладка дозволяє користувачам налаштовувати категорію витрат, якій відповідає працівник протягом певного періоду. Ця інформація використовується для розрахунку витрат, пов'язаних із працівником на проєкті.

    .. figure:: images/worker-costcategory.png
       :scale: 50

       Вкладка категорії витрат для ресурсу

Призначення ресурсів пояснюється в розділі «Призначення ресурсів».

Управління машинами
====================

Машини розглядаються як ресурси для всіх цілей. Тому, подібно до працівників, машинами можна управляти та призначати їх до завдань. Призначення ресурсів розглядається в розділі «Призначення ресурсів», де будуть пояснені специфічні особливості машин.

Машинами управляють через пункт меню «Ресурси». Цей розділ має операцію «Список машин», яка відображає машини компанії. Користувачі можуть редагувати або видаляти машину зі списку.

При редагуванні машин система відображає ряд вкладок для управління різними деталями:

1.  **Дані машини:** Ця вкладка дозволяє редагувати ідентифікаційні дані машини:

    i.  Назва
    ii. Код машини
    iii. Опис машини

    .. figure:: images/machine-data.png
       :scale: 50

       Редагування даних машини

2.  **Критерії:** Як і у випадку з ресурсами-працівниками, ця вкладка використовується для додавання критеріїв, яким відповідає машина. Машинам можна призначати два типи критеріїв: специфічні для машин або загальні. Критерії для працівників не можна призначати машинам. Щоб призначити критерії:

    i.  Натисніть кнопку «Додати критерії».
    ii. Знайдіть критерій для додавання та виберіть найбільш відповідний.
    iii. Виберіть дату початку застосування критерію.
    iv. Виберіть дату закінчення застосування критерію до ресурсу. Ця дата є необов'язковою, якщо критерій вважається безстроковим.
    v.  Натисніть кнопку «Зберегти і продовжити».

    .. figure:: images/machine-criterions.png
       :scale: 50

       Призначення критеріїв машинам

3.  **Календар:** Ця вкладка дозволяє користувачам налаштовувати специфічний календар для машини. Усім машинам призначено стандартний календар; однак можна призначити специфічний календар кожній машині на основі наявного.

    .. figure:: images/machine-calendar.png
       :scale: 50

       Призначення календарів машинам

4.  **Конфігурація машини:** Ця вкладка дозволяє користувачам налаштовувати відношення машин до ресурсів-працівників. Машина має значення альфа, що вказує відношення машина/працівник. Як уже згадувалося, значення альфа 0,5 означає, що для кожного повного робочого дня машини потрібно 0,5 особи. На основі значення альфа система автоматично призначає працівників, пов'язаних із машиною, після призначення машини до завдання. Пов'язування працівника з машиною може здійснюватися двома способами:

    i.  **Конкретне призначення:** Призначте діапазон дат, протягом яких працівник призначений для машини. Це конкретне призначення, оскільки система автоматично призначає години працівнику при плануванні машини.
    ii. **Загальне призначення:** Призначте критерії, яким повинні відповідати працівники, призначені до машини. Це створює загальне призначення працівників, які відповідають критеріям.

    .. figure:: images/machine-configuration.png
       :scale: 50

       Конфігурація машин

5.  **Категорія витрат:** Ця вкладка дозволяє користувачам налаштовувати категорію витрат, якій відповідає машина протягом певного періоду. Ця інформація використовується для розрахунку витрат, пов'язаних із машиною на проєкті.

    .. figure:: images/machine-costcategory.png
       :scale: 50

       Призначення категорій витрат машинам

Віртуальні групи працівників
=============================

Програма дозволяє користувачам створювати віртуальні групи працівників, які не є реальними працівниками, але є змодельованим персоналом. Ці групи дозволяють користувачам моделювати збільшення виробничої потужності в конкретні часи на основі налаштувань календаря.

Віртуальні групи працівників дозволяють користувачам оцінювати, як на планування проєкту вплине найм і призначення персоналу, який відповідає конкретним критеріям, тим самим допомагаючи в прийнятті рішень.

Вкладки для створення віртуальних груп працівників ті самі, що й для налаштування працівників:

*   Загальні дані
*   Призначені критерії
*   Календарі
*   Пов'язані години

Різниця між віртуальними групами працівників і реальними працівниками полягає в тому, що у віртуальних груп є назва групи та кількість, яка представляє кількість реальних людей у групі. Є також поле для коментарів, де можна надати додаткову інформацію, наприклад, для якого проєкту буде потрібний найм, еквівалентний віртуальній групі працівників.

Ресурси на основі черги
========================

Ресурси на основі черги — це специфічний тип продуктивного елемента, який може бути або непризначеним, або мати 100% завантаження. Іншими словами, вони не можуть мати більше одного завдання, запланованого одночасно, і не можуть бути перевантажені.

Для кожного ресурсу на основі черги автоматично створюється черга. Завданнями, запланованими для цих ресурсів, можна управляти конкретно за допомогою наданих методів призначення, створюючи автоматичні призначення між завданнями та чергами, що відповідають необхідним критеріям, або переміщуючи завдання між чергами.
