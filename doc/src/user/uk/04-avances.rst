Прогрес
########

.. contents::

Прогрес проєкту вказує на ступінь виконання розрахункового часу завершення проєкту. Прогрес завдання вказує на ступінь виконання завдання відповідно до його розрахункового завершення.

Загалом прогрес неможливо виміряти автоматично. Досвідчений працівник або чек-лист повинні визначити ступінь завершеності завдання або проєкту.

Важливо зазначити різницю між годинами, призначеними завданню або проєкту, та прогресом цього завдання або проєкту. Хоча кількість використаних годин може бути більше або менше очікуваної, проєкт може випереджати або відставати від розрахункового завершення на день моніторингу. З цих двох вимірювань можуть виникнути кілька ситуацій:

*   **Витрачено менше годин, ніж очікувалося, але проєкт відстає від графіка:** Прогрес нижчий за запланований на день моніторингу.
*   **Витрачено менше годин, ніж очікувалося, і проєкт випереджає графік:** Прогрес вищий за запланований на день моніторингу.
*   **Витрачено більше годин, ніж очікувалося, і проєкт відстає від графіка:** Прогрес нижчий за запланований на день моніторингу.
*   **Витрачено більше годин, ніж очікувалося, але проєкт випереджає графік:** Прогрес вищий за запланований на день моніторингу.

Вид планування дозволяє порівнювати ці ситуації, використовуючи інформацію про досягнутий прогрес та використані години. У цьому розділі пояснюється, як вводити інформацію для моніторингу прогресу.

Філософія моніторингу прогресу заснована на тому, що користувачі визначають рівень, на якому хочуть стежити за своїми проєктами. Наприклад, якщо користувачі хочуть стежити за проєктами, їм потрібно лише вводити інформацію для елементів першого рівня. Якщо вони хочуть більш точного моніторингу на рівні завдань, вони повинні вводити інформацію про прогрес на нижчих рівнях. Система потім агрегує дані вгору через ієрархію.

Управління типами прогресу
============================

Компанії мають різні потреби при моніторингу прогресу проєкту, зокрема залучених завдань. Тому система включає «типи прогресу». Користувачі можуть визначати різні типи прогресу для вимірювання прогресу завдання. Наприклад, завдання може вимірюватися у відсотках, але цей відсоток може також перетворюватися на прогрес у *тоннах* на основі угоди з клієнтом.

Тип прогресу має назву, максимальне значення та значення точності:

*   **Назва:** Описова назва, яку користувачі впізнаватимуть при виборі типу прогресу. Ця назва повинна чітко вказувати, який вид прогресу вимірюється.
*   **Максимальне значення:** Максимальне значення, яке може бути встановлено для завдання або проєкту як загальне вимірювання прогресу. Наприклад, якщо ви працюєте з *тоннами* і звичайний максимум становить 4000 тонн, і жодне завдання ніколи не вимагатиме більше 4000 тонн будь-якого матеріалу, то 4000 буде максимальним значенням.
*   **Значення точності:** Значення кроку, дозволене для типу прогресу. Наприклад, якщо прогрес у *тоннах* вимірюватиметься цілими числами, значення точності дорівнюватиме 1. Починаючи з цього моменту, як вимірювання прогресу можна вводити лише цілі числа (наприклад, 1, 2, 300).

Система має два стандартних типи прогресу:

*   **Відсоток:** Загальний тип прогресу, що вимірює прогрес проєкту або завдання на основі розрахункового відсотка завершення. Наприклад, завдання виконано на 30% із 100%, запланованих на конкретний день.
*   **Одиниці:** Загальний тип прогресу, що вимірює прогрес в одиницях без вказівки типу одиниці. Наприклад, завдання передбачає створення 3000 одиниць, а прогрес становить 500 одиниць із загальних 3000.

.. figure:: images/tipos-avances.png
   :scale: 50

   Адміністрування типів прогресу

Користувачі можуть створювати нові типи прогресу таким чином:

*   Перейдіть до розділу «Адміністрування».
*   Натисніть параметр «Управління типами прогресу» в меню другого рівня.
*   Система відобразить список наявних типів прогресу.
*   Для кожного типу прогресу користувачі можуть:

    *   Редагувати
    *   Видалити

*   Потім користувачі можуть створити новий тип прогресу.
*   При редагуванні або створенні типу прогресу система відображає форму з такою інформацією:

    *   Назва типу прогресу.
    *   Максимально допустиме значення для типу прогресу.
    *   Значення точності для типу прогресу.

Введення прогресу за типом
============================

Прогрес вводиться для елементів проєкту, але його також можна вводити за допомогою ярлика з планувальних завдань. Користувачі відповідають за вирішення, який тип прогресу пов'язувати з кожним елементом проєкту.

Користувачі можуть ввести єдиний стандартний тип прогресу для всього проєкту.

Перш ніж вимірювати прогрес, користувачі повинні пов'язати вибраний тип прогресу з проєктом. Наприклад, вони можуть вибрати відсотковий прогрес для вимірювання прогресу всього завдання або узгоджений показник прогресу, якщо в майбутньому будуть вводитися вимірювання прогресу, узгоджені з клієнтом.

.. figure:: images/avance.png
   :scale: 40

   Екран введення прогресу з графічною візуалізацією

Щоб ввести вимірювання прогресу:

*   Виберіть тип прогресу, до якого буде додано прогрес.
    *   Якщо типу прогресу не існує, потрібно створити новий.
*   У формі, що з'явиться під полями «Значення» та «Дата», введіть абсолютне значення вимірювання та дату вимірювання.
*   Система автоматично зберігає введені дані.

Порівняння прогресу для елемента проєкту
==========================================

Користувачі можуть графічно порівнювати прогрес, досягнутий у проєктах, із зробленими вимірюваннями. Усі типи прогресу мають стовпець із кнопкою перемикача («Показати»). Коли цю кнопку вибрано, для елемента проєкту відображається графік прогресу зроблених вимірювань.
