Інформаційна панель проєкту
############################

.. contents::

Інформаційна панель проєкту — це перспектива *LibrePlan*, що містить набір **КПЕ (ключових показників ефективності)** для оцінки ефективності проєкту з точки зору:

   *   Прогресу виконання робіт
   *   Вартості
   *   Стану розподілених ресурсів
   *   Часових обмежень

Індикатори прогресу виконання
==============================

Розраховуються два індикатори: відсоток прогресу проєкту та статус завдань.

Відсоток прогресу проєкту
--------------------------

Цей графік відображає загальний прогрес проєкту, порівнюючи його з очікуваним прогресом на основі діаграми *Ганта*.

Прогрес відображається двома смугами:

   *   *Поточний прогрес:* Поточний прогрес на основі зроблених вимірювань.
   *   *Очікуваний прогрес:* Прогрес, якого проєкт повинен був досягти на цьому етапі відповідно до плану проєкту.

Щоб переглянути фактичне виміряне значення для кожної смуги, наведіть курсор миші на смугу.

Загальний прогрес проєкту оцінюється кількома різними методами, оскільки не існує єдиного загальноприйнятого підходу:

   *   **Поширений прогрес:** Це тип прогресу, встановлений як поширений прогрес на рівні проєкту. У цьому випадку немає способу розрахувати очікуване значення, тому відображається лише поточна смуга.
   *   **За всіма годинами завдань:** Прогрес всіх завдань проєкту усереднюється для розрахунку загального значення. Це зважене середнє, яке враховує кількість годин, виділених для кожного завдання.
   *   **За годинами критичного шляху:** Прогрес завдань, що належать до будь-якого критичного шляху проєкту, усереднюється для отримання загального значення. Це зважене середнє, яке враховує загальну кількість виділених годин для кожного залученого завдання.
   *   **За тривалістю критичного шляху:** Прогрес завдань, що належать до будь-якого критичного шляху, усереднюється за допомогою зваженого середнього, але цього разу з урахуванням тривалості кожного залученого завдання замість призначених годин.

Статус завдань
--------------

Кругова діаграма показує відсоток завдань проєкту в різних станах. Визначені стани:

   *   **Завершені:** Виконані завдання, визначені значенням прогресу 100%.
   *   **В процесі виконання:** Завдання, що виконуються наразі. Ці завдання мають значення прогресу, що не дорівнює 0% або 100%, або для них відстежено певний робочий час.
   *   **Готові до початку:** Завдання з 0% прогресу, без відстеженого часу, де всі залежні завдання типу *FINISH_TO_START* є *завершеними*, а всі залежні завдання типу *START_TO_START* є *завершеними* або *в процесі виконання*.
   *   **Заблоковані:** Завдання з 0% прогресу, без відстеженого часу, з попередніми залежними завданнями, що не є ні *в процесі виконання*, ні в стані *готові до початку*.

Вартісні індикатори
====================

Розраховуються кілька вартісних індикаторів *управління освоєним обсягом*:

   *   **CV (Відхилення вартості):** Різниця між кривою *Освоєного обсягу* та кривою *Фактичної вартості* в поточний момент. Позитивні значення вказують на прибуток, негативні — на збиток.
   *   **ACWP (Фактична вартість виконаних робіт):** Загальна кількість годин, відстежених у проєкті в поточний момент.
   *   **CPI (Індекс виконання вартості):** Співвідношення *Освоєний обсяг / Фактична вартість*.

        *   > 100 сприятливо — проєкт виконується в межах бюджету.
        *   = 100 також сприятливо — вартість відповідає плану.
        *   < 100 несприятливо — вартість виконання роботи вища від запланованої.
   *   **ETC (Розрахункова вартість до завершення):** Час, що залишився до завершення проєкту.
   *   **BAC (Бюджет при завершенні):** Загальний обсяг робіт, виділений у плані проєкту.
   *   **EAC (Розрахункова вартість при завершенні):** Прогноз менеджера щодо загальної вартості при завершенні проєкту на основі *CPI*.
   *   **VAC (Відхилення при завершенні):** Різниця між *BAC* та *EAC*.

        *   < 0 вказує, що проєкт виконується з перевищенням бюджету.
        *   > 0 вказує, що проєкт виконується в межах бюджету.

Ресурси
========

Для аналізу проєкту з точки зору ресурсів надаються два коефіцієнти та гістограма.

Гістограма відхилення оцінки для завершених завдань
----------------------------------------------------

Ця гістограма розраховує відхилення між кількістю годин, виділених для завдань проєкту, та фактичною кількістю годин, присвячених їм.

Відхилення розраховується у відсотках для всіх завершених завдань, а розраховані відхилення представлені у вигляді гістограми. Вертикальна вісь показує кількість завдань у кожному інтервалі відхилення. Динамічно розраховуються шість інтервалів відхилення.

Коефіцієнт понаднормових
-------------------------

Цей коефіцієнт підсумовує перевантаженість ресурсів, виділених для завдань проєкту. Він розраховується за формулою: **коефіцієнт понаднормових = перевантаження / (завантаження + перевантаження)**.

   *   = 0 сприятливо — ресурси не перевантажені.
   *   > 0 несприятливо — ресурси перевантажені.

Коефіцієнт доступності
-----------------------

Цей коефіцієнт підсумовує вільну потужність ресурсів, наразі виділених для завдань проєкту. Тому він вимірює доступність ресурсів для отримання додаткових призначень без перевантаження. Розраховується як: **коефіцієнт доступності = (1 - завантаження/потужність) * 100**

   *   Можливі значення від 0% (повністю призначено) до 100% (не призначено).

Час
====

Включено два графіки: гістограма відхилення часу завершення завдань проєкту та кругова діаграма порушень кінцевих строків.

Випередження або відставання у виконанні завдань
-------------------------------------------------

Цей розрахунок визначає різницю в днях між запланованим часом завершення завдань проєкту та їхнім фактичним часом завершення. Запланована дата завершення береться з діаграми *Ганта*, а фактична дата завершення — з останнього відстеженого часу для завдання.

Відставання або випередження у виконанні завдань представлено у вигляді гістограми. Вертикальна вісь показує кількість завдань зі значенням відхилення в днях, що відповідає інтервалу днів на горизонтальній осі. Динамічно розраховуються шість інтервалів відхилення завершення завдань.

   *   Від'ємні значення означають завершення раніше графіка.
   *   Позитивні значення означають завершення пізніше графіка.

Порушення кінцевих строків
---------------------------

У цьому розділі розраховується запас до кінцевого строку проєкту (якщо встановлено). Крім того, кругова діаграма показує відсоток завдань, що дотримуються свого кінцевого строку. До діаграми включено три типи значень:

   *   Відсоток завдань без налаштованого кінцевого строку.
   *   Відсоток завершених завдань з фактичною датою завершення пізніше кінцевого строку. Фактична дата завершення береться з останнього відстеженого часу для завдання.
   *   Відсоток завершених завдань з фактичною датою завершення раніше кінцевого строку.
