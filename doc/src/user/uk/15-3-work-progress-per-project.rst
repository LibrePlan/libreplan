Звіт про роботи та прогрес за проєктом
########################################

.. contents::

Призначення
============

Цей звіт надає огляд стану проєктів з урахуванням як прогресу, так і вартості.

Він аналізує поточний прогрес кожного проєкту, порівнюючи його із запланованим прогресом та виконаними роботами.

Звіт також відображає кілька показників, пов'язаних із вартістю проєкту, порівнюючи поточну ефективність із запланованою.

Вхідні параметри та фільтри
============================

Є кілька обов'язкових параметрів:

   *   **Дата відліку:** Це дата, що використовується як точка відліку для порівняння запланованого стану проєкту з фактичними показниками. *Стандартним значенням для цього поля є поточна дата*.

   *   **Тип прогресу:** Це тип прогресу, що використовується для вимірювання прогресу проєкту. Застосунок дозволяє вимірювати проєкт одночасно різними типами прогресу. Тип, вибраний користувачем у спадному меню, використовується для розрахунку даних звіту. Стандартним значенням *типу прогресу* є *spread* — спеціальний тип прогресу, що використовує бажаний метод вимірювання прогресу, налаштований для кожного елемента ІСР.

Необов'язкові параметри:

   *   **Дата початку:** Це найрання дата початку проєктів, що включаються до звіту. Якщо це поле залишити порожнім, мінімальна дата початку для проєктів відсутня.

   *   **Дата завершення:** Це найпізніша дата завершення проєктів, що включаються до звіту. Всі проєкти, що завершуються після *Дати завершення*, будуть виключені.

   *   **Фільтр за проєктами:** Цей фільтр дозволяє користувачам вибирати конкретні проєкти для включення до звіту. Якщо до фільтра не додано жодних проєктів, звіт включатиме всі проєкти в базі даних. Для пошуку потрібного проєкту надається спадне меню з можливістю пошуку. Проєкти додаються до фільтра натисканням кнопки *Додати*.

Вихідні дані
=============

Формат виводу такий:

Заголовок
----------

У заголовку звіту відображаються такі поля:

   *   **Дата початку:** Дата фільтрації початку. Не відображається, якщо звіт не фільтрується за цим полем.
   *   **Дата завершення:** Дата фільтрації завершення. Не відображається, якщо звіт не фільтрується за цим полем.
   *   **Тип прогресу:** Тип прогресу, що використовується у звіті.
   *   **Проєкти:** Вказує відфільтровані проєкти, для яких формується звіт. Відображатиметься рядок *Всі*, коли звіт включає всі проєкти, що задовольняють іншим фільтрам.
   *   **Дата відліку:** Обов'язкова вхідна дата відліку, вибрана для звіту.

Нижній колонтитул
------------------

У нижньому колонтитулі відображається дата формування звіту.

Тіло
-----

Тіло звіту складається зі списку проєктів, вибраних на основі вхідних фільтрів.

Фільтри працюють шляхом додавання умов, за винятком набору, утвореного фільтрами дат (*Дата початку*, *Дата завершення*) та *Фільтром за проєктами*. У цьому випадку, якщо один або обидва фільтри дат заповнені і *Фільтр за проєктами* містить список вибраних проєктів, останній фільтр має пріоритет. Це означає, що проєкти, включені до звіту, — це проєкти, надані *Фільтром за проєктами*, незалежно від фільтрів дат.

Важливо зазначити, що прогрес у звіті розраховується як частка від одиниці в діапазоні від 0 до 1.

Для кожного проєкту, вибраного для включення до виводу звіту, відображається така інформація:

   * *Назва проєкту*.
   * *Загальна кількість годин*. Загальна кількість годин для проєкту показується шляхом додавання годин для кожного завдання. Показуються два типи загальної кількості годин:
      *   *Розрахункові (TE)*. Це сума всіх розрахункових годин у ІСР проєкту. Вона представляє загальну кількість годин, розрахованих для завершення проєкту.
      *   *Заплановані (TP)*. В *LibrePlan* можливі два різних значення: розрахункова кількість годин для завдання (кількість годин, спочатку розрахована для виконання завдання) та заплановані години (години, виділені в плані для виконання завдання). Заплановані години можуть дорівнювати, бути меншими або більшими за розрахункові, і визначаються на пізнішому етапі — операції призначення. Тому загальна кількість запланованих годин для проєкту є сумою всіх виділених годин для його завдань.
   * *Прогрес*. Показуються три вимірювання, пов'язані із загальним прогресом типу, вказаного у фільтрі вхідних даних прогресу для кожного проєкту на дату відліку:
      *   *Виміряний (PM)*. Це загальний прогрес з урахуванням вимірювань прогресу з датою раніше *Дати відліку* у вхідних параметрах звіту. Враховуються всі завдання, і сума зважується за кількістю годин для кожного завдання.
      *   *Приписаний (PI)*. Це прогрес, що передбачає, що робота продовжується з тим самим темпом, що й завершені для завдання години. Якщо завершено X годин із Y годин для завдання, загальний приписаний прогрес вважається рівним X/Y.
      *   *Запланований (PP)*. Це загальний прогрес проєкту відповідно до запланованого розкладу на дату відліку. Якщо все відбувалося точно за планом, виміряний прогрес повинен збігатися із запланованим прогресом.
   * *Години до дати*. Є два поля, що показують кількість годин до дати відліку з двох точок зору:
      *   *Заплановані (HP)*. Це число є сумою годин, виділених для будь-якого завдання в проєкті з датою меншою або рівною *Даті відліку*.
      *   *Фактичні (HR)*. Це число є сумою годин, зазначених у звітах про роботу для будь-якого завдання проєкту з датою меншою або рівною *Даті відліку*.
   * *Різниця*. Під цим заголовком є кілька показників, пов'язаних із вартістю:
      *   *Вартість*. Це різниця в годинах між кількістю витрачених годин з урахуванням виміряного прогресу та годинами, виконаними до дати відліку. Формула: *PM*TP - HR*.
      *   *Запланована*. Це різниця між годинами, витраченими відповідно до загального виміряного прогресу проєкту, та кількістю, запланованою до *Дати відліку*. Вимірює перевагу або відставання в часі. Формула: *PM*TP - HP*.
      *   *Коефіцієнт вартості*. Розраховується шляхом ділення *PM* / *PI*. Якщо він більше 1, це означає, що проєкт є прибутковим у цій точці. Якщо він менше 1, це означає, що проєкт збитковий.
      *   *Коефіцієнт розкладу*. Розраховується шляхом ділення *PM* / *PP*. Якщо він більше 1, це означає, що проєкт випереджає графік. Якщо він менше 1, це означає, що проєкт відстає від графіка.
