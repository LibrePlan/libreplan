Планувальник
#############

.. contents::

Планувальник призначений для динамічного планування завдань. Він розроблений з використанням *планувальника Spring Framework Quartz*.

Щоб ефективно використовувати цей планувальник, спочатку необхідно створити завдання (завдання Quartz), які потрібно планувати. Потім ці завдання можна додати до бази даних, оскільки всі завдання для планування зберігаються в базі даних.

Коли планувальник запускається, він зчитує завдання для планування або скасування планування з бази даних та відповідно планує або видаляє їх. Після цього завдання можна динамічно додавати, оновлювати або видаляти за допомогою інтерфейсу ``Планування завдань``.

.. NOTE::
   Планувальник запускається при запуску вебзастосунку LibrePlan і зупиняється при зупинці застосунку.

.. NOTE::
   Цей планувальник підтримує лише ``вирази cron`` для планування завдань.

Критерії, які планувальник використовує для планування або видалення завдань при запуску:

Для всіх завдань:

* Планування

  * Завдання має *Конектор*, і *Конектор* активований, і завдання дозволено планувати.
  * Завдання не має *Конектора* і дозволено планувати.

* Видалення

  * Завдання має *Конектор*, і *Конектор* не активований.
  * Завдання має *Конектор*, і *Конектор* активований, але завдання не дозволено планувати.
  * Завдання не має *Конектора* і не дозволено планувати.

.. NOTE::
   Завдання не можна перепланувати або скасувати плановане, якщо вони наразі виконуються.

Вид списку планування завдань
==============================

Вид ``Список планування завдань`` дозволяє користувачам:

*   Додавати нове завдання.
*   Редагувати наявне завдання.
*   Видаляти завдання.
*   Запускати процес вручну.

Додавання або редагування завдання
===================================

У виді ``Список планування завдань`` натисніть:

*   ``Створити`` для додавання нового завдання, або
*   ``Редагувати`` для зміни вибраного завдання.

Обидві дії відкриють форму створення/редагування ``завдання``. ``Форма`` відображає такі властивості:

*   Поля:

    *   **Група завдань:** Назва групи завдань.
    *   **Назва завдання:** Назва завдання.
    *   **Вираз cron:** Поле лише для читання з кнопкою ``Редагувати`` для відкриття вікна введення ``виразу cron``.
    *   **Назва класу завдання:** ``Спадний список`` для вибору завдання (наявного завдання).
    *   **Конектор:** ``Спадний список`` для вибору конектора. Не є обов'язковим.
    *   **Розклад:** Прапорець для вказівки, чи планувати це завдання.

*   Кнопки:

    *   **Зберегти:** Для збереження або оновлення завдання як у базі даних, так і в планувальнику. Після цього користувач повертається до ``Виду списку планування завдань``.
    *   **Зберегти і продовжити:** Те саме, що «Зберегти», але користувач не повертається до ``Виду списку планування завдань``.
    *   **Скасувати:** Нічого не зберігається, і користувач повертається до ``Виду списку планування завдань``.

*   І розділ підказки про синтаксис виразу cron.

Спливаюче вікно виразу cron
-----------------------------

Для правильного введення ``виразу cron`` використовується спливаюча форма ``виразу cron``. У цій формі можна ввести потрібний ``вираз cron``. Дивіться також підказку про ``вираз cron``. Якщо ввести недійсний ``вираз cron``, ви отримаєте негайне сповіщення.

Видалення завдання
===================

Натисніть кнопку ``Видалити``, щоб видалити завдання як із бази даних, так і з планувальника. Успіх або невдача цієї дії відобразяться.

Запуск завдання вручну
=======================

Як альтернатива очікуванню виконання завдання за розкладом, можна натиснути цю кнопку для безпосереднього запуску процесу. Після цього інформація про успіх або невдачу відобразиться у ``спливаючому вікні``.
